[5 장] - 프로세스

프로세스

  • 메모리에 로드되어 실행중인 프로그램. - 모호함

프로세스 구체적 이해

  • 프로세스 범위
    • 메모리 구조(가상) + 레지스터 set
    • 메모리에서 레지스터로 옮겨진 후 연산이 되는 것이 프로세스 동작이므로.
    • 다른 프로세스의 연산을 위해 점유하고 있던 레지스터 set 을 비우는데 이것이 바로 컨텍스트 스위칭.
    • 더 정확한 정의이다.

프로세스 스케줄러

  • 스케줄러
    • 프로세스를 스케줄링 해주기 위한 sw 적인 장치. 블록.
    • 일종의 별도 프로세스.
  • 스케줄러 기능
    • 둘 이상의 프로세스가 절절히 실행되도록 컨트롤
  • 스케줄링 방법
    • 스케줄링 알고리즘에 따라 다양함.

프로세스 상태

  • Ready
    • 메모리만 할당받고 CPU 는 할당받지 못한 상태.
    • 스케줄러만이 Running 상태로 바꿔줄 수 있다.
    • 실행하면 처음에 Ready 가 된다.
  • Running
    • CPU 할당을 받아 실행중인 상태.
    • 다른 프로세스가 선점하도록 하기위해 Ready 상태로 내려가도록 하기도 한다.
  • Blocked
    • I/O 연산은 CPU 에 의존적이지 않다.
    • 그래서 기타 CPU 의존적 연산과 병행할 수 있다.
    • I/O 처리를 하는 연산은 I/O 작업이 끝날때까지 스케줄러가 Block 상태에 머무르게 한다.
    • I/O 끝나면 Ready 상태로 된다.

컨텍스트 스위칭

  • 실행중인 프로세스의 PCB(레지스터 set, pid 등등) 을 메모리로 내리면 save
  • 메모리에 있던 PCB 를 CPU 로 올리면 restore
  • 둘을 합치면 스위칭.
  • 빈번하게 일어난다.

프로세스 생성방법

  • 실행파일을 실행함으로서 프로세스 생성
    • 바탕화면도 프로세스이므로 부모 프로세스로 볼 수 있다.
  • 프로그래밍으로 프로세스 생성
    • 부모 프로세스에서 자식 프로세스를 생성
  • 시스템 데몬처럼 시스템이 생성하는 프로세스
  • 동일한 프로세스의 개념이다.

프로세스 생성 함수
