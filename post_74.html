알고리즘 강의노트

병합정렬까지

연습방법

잘 작성된 알고리즘을 이해하고 스스모 만들어봐야 한다.
무엇이든지 copy 로 시작한다. 가장 빠르고 효율적인 방법이다.
항상 지면에 시뮬레이션을 한다.
알고리즘은 종이에 다 만들고 옮기는것만 코드로 구현한다.
기본 정렬 3 가지는 매우 쉬우니까 그냥 숙지하고 있으면 된다.

버블 정렬

상향식, 하향식 둘 다 작성 가능.
스왑이 일어나지 않았을 때 조기종료하여 최적화 가능.

선택 정렬

최대 최소를 찾아서 옮기는 알고리즘

삽입 정렬

하나씩 빼서 새로운 자리에 끼워넣는다.

공간 복잡도

시간 복잡도와 대체적으로 반비례 성격.
좀 오래되었지만 면접에서 물어볼 때도 있음.
반복문에선 변수가 새로 만들 필요없는 것도 재귀함수를 쓰면 인자 변수가 계속 생성되는 경우가 있다.

재귀

역시 지면에 먼저 작성하는 것이 실수를 줄일 수 있다.
반복문으로 그대로 변환할 수 있으며 시간복잡도는 동일하다.

DP 와 분할정복

둥 다 문제를 잘게 쪼개서 최소 단위로 분할한다.
DP 는 작은 문제가 중복되는 경우에 적용하지만 분할 정복은 중복이 안 일어난다.
그래서 DP 는 메모리가 필요하고 분할 정복은 별도의 메모리가 필요없다.
DP 는 반복문/재귀 둘 다 활용 가능하다.

퀵정렬

정렬의 꽃
재귀를 이용한 분할정복
pivot 을 정하고 보다 큰 것, 작은것들로 나눔.
나눈 것들에 대해 재귀적으로 반복한다.
이들을 합치면 그대로 완전히 정렬이 된다.
파이썬 표현식으로 매우 단순하게 작성이 가능하다.

퀵정렬 시간복잡도

평균적으로 O(nlogn). 각 단계마다 n 번, 총 logn 단계
left, right 가 절반 가까이 씩 될때
최악의 경우 O(n^2). pivot 이 최대값, 최소값일 때.
총 단계가 n 단계가 되므로 n 제곱이 됨.

병합 정렬

분할 정복을 사용. 그러므로 동적계획법은 아님.
절반으로 계속 쪼개고 각각 합친다.
재귀함수는 절반으로 쪼갠것을 다시 합치는 메커니즘.

병합 정렬 시간 복잡도

퀵소트처럼 각 단계는 O(n) 총 단계는 logn 개 결과적으로 O(nlogn)
