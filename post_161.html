파일 시스템

  • 사용자들이 사용하는 파일들을 관리하는 OS 의 한 부분.
  • 파일 시스템의 구성
    1. files
    2. directory structure
    3. partitions

File Concept

  • 보조 기억장치(디스크)에 저장된, 연관된 정보들의 집합
    디스크 할당의 최소 단위
    sequence of bytes (물리적 정의)
  • 내용에 따른 분류
    program file
    data file
  • 형태의 따른 분류
    text tile
    binary file
  • file attributes (속성)
    name, identifier, type, size, protection 등
  • file operations
    create, write, read, reposition, delete 등
    OS 는 file operation 들에 대한 sytem call 을 제공해야한다!

File Access Methods

  • sequential access - 순차 접근
    파일을 record, 혹은 bytes 단위로 순서대로 접근
  • directed access - 직접 접근
    원하는 블럭을 직접 접근
  • indexed access
    index (목차) 를 참조하여 원하는 블럭을 찾은 후 데이터에 접근
    배열의 인덱스를 지정하여 접근하는 경우

File system organization

  • partitions
    물리적으로는 디스크가 하나라도 여러개 나눠서 쓸 수 있다.
    반대로 여러개를 합쳐서 하나의 디스크처럼 쓰는 것도 파티션 개념이다.
    mini disk, volumes
    가상 디스크
  • directory
    파일들을 분류, 보관하기 위한 개념
    디렉토리를 위한 연산 제공
    파일 검색, 생성, 삭제, 리스트, 이름 변경, 탐색 등
    OS 가 system call 을 통해 제공해줘야 한다.

Mounting

  • 현재 file system 에 다른 file system 에 붙이는 것.
  • 안드로이드 등에 /mnt 폴더가 있다.
  • sd 카드를 추가하고 접근할 때 등에 해당한다.

디렉토리

  • 논리적인 디렉토리 구조

Flat Directory Structure

  • 파일 시스템 내에 하나의 디렉토리만 존재
    single level - 시스템에 디렉토리가 하나만 있음.
  • 문제점
    file naming - 이름 충돌 때문에 네이밍이 어려움.
    file protection
    file managment
    다중 사용자 환경에서 문제가 더욱 커짐
    초기 시스템의 형태

2-Level Directory Structure

  • 사용자 마다 하나의 디렉토리 배정, 2 계층 구조
  • 구조
    MFD - 마스터 파일 디렉토리
    UFD - 유저 파일 디렉토리
  • 문제점
    sub directory 생성 불가능 - 파일 네이밍 문제
    사용자간 파일 공유 불가

Hierarchical directory structure

  • tree 형태의 계층적 디렉토리 사용 가능
  • 사용자가 하부 디렉토리 생성/관리 가능
    system call 이 제공되엉야 ㅎ마.
    용어가 새로 등장 - 홈 디렉, 현재 디렉, 절대 경로, 상대 경로 등.
  • 대부분의 OS 가 사용

Acyclic Graph Directory Structure

  • 계층적 디렉토리 구조의 확장
  • 사이클이 발생하지 않는 구조이다.
  • 디렉토리 안에 공유 디렉토리, 공유 파일을 담을 수 있음.
  • Link 의 개념 사용.
    바로가기 등.
    symbolic link

General Graph Directory Structure

  • 사이클을 허용하는 디렉토리 구조
  • 보통 상위 계층에 올라갈 때 사이클이 생길 수 있다.
  • 문제점
    파일 탐색시 무한루프가 생길 수 있다.
