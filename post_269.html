시스템 프로그래밍

  • 운영체제에서 제공하는 강력한 기능들을 사용하기 위해 SP 를 배운다.
  • 모든 응용 프로그램에 사용되는 핵심적인 것.
  • 시스템에서 동작하는 프로그램.
  • 시스템 프로그래밍은 운영체제에 의존적임.

[1 장] 시스템 프로그래밍 이해와 접근 / 하드웨어 구성

시스템 프로그래밍이란?

  • 시스템(컴 시스템)의 범위
    • 하드웨어 + 운영체제 둘의 묶음으로 본다.
  • 시스템 프로그래밍
    • 시스템 즉, 하드웨어와 운영체제의 기능을 활용하는 프로그래밍
    • 모든 응용 프로그램은 시스템을 어느정도 활용해야 한다.
  • c 언어를 예로 들면 언어차원에서 국한되지 않고 시스템, 커널을 제어할 수 있는 기능을 사용하는 것!

컴구조 간단

  • 시스템
    • CPU - 캐시 - 메모리 - 디스트
  • 컴퓨터 구조
    • CPU, 캐시 까지
  • 운영체제
    • RAM, 파일, 입출력 까지

하드웨어 구성

  1. CPU
    • 연산이 이루어지는 원리
  2. 메인 메모리
    • 프로그램 실행 방식 이해
  3. 입출력 버스
    • 데이터 송수신이 이루어지는 원리
  • 하드웨어 구성요소들을 묘사한 것은 매우 추상적인 것임.
  • 그림은 개념적 이해를 위한 도구이다.

CPU

  • ALU
    • 실제 연산을 수행함.
    • 컨트롤 유닛의 제어를 받아서 수행.
    • 레지스터에서 피연산자를 가져온다.
  • 컨트롤 유닛
    • 모든 요소들과 신호를 주고 받는다.
    • 레지스터의 명령어를 읽고 판단한다.
    • 두뇌 역할
  • 레지스터
    • 버스 인터페이스를 통해 들어온 명령어들이 저장된다.
  • 버스 인터페이스
    • 다양한 장치들로부터 들어오는 데이터 및 신호들을 받아들인다.

클럭 신호

  • 동작 타이밍
    • 클럭 발생기의 클럭
    • 모든 장치들은 타이밍을 맞추어 동기화됨.
    • 없으면 데이터 분실 등이 발생할 수 있음.

명령어

  • CPU 디자인하면서 명령어도 함께 한다.
  • 명령어에 따라 CPU 동작이 달라진다.

프로그램 실행과정

  • 전처리기에 의한 치환작업
    • # 으로 시작하는 지시자
  • 컴파일러에 의한 번역
    • CPU 명령어로 번역
  • 어셈블러에 의한 바이너리 코드 생성
    • binary 코드로 번역
  • 링커에 의한 연결과 결합
    • 라이브러리와의 결합
    • 실행파일을 만드는 것

Stored Program Concept

  • 프로그램을 메모리에 내장하게 되는 아키텍처
  • 명령어를 메모리에 올리는 방식.
  • 아래 작업은 CPU 가 주체.
  1. Fetch
    • 메모리에서 CPU 내부로 명령어 이동
    • 버스, 버스 인터페이스에 의해서
  2. Decode
    • 명령어 해석
    • 컨트롤 유닛
  3. Execution
    • 연산을 진행
    • 보통은 ALU. 구체적으로는 여러 요소들이 협력해서 EXECUTION

Stored Program Concept + 컴퓨터 구조

  • 메모리
    • 디스크에 저장되었던 코드들이 프로그램으로 메모리에 올라옴.
    • 컴파일된(컴파일 언어일 때) 바이너리/명령어가 올라옴.
  • 버스
    • 이 명령어들이 버스를 타고 CPU 로 이동한다.
  • CPU
    • 레지스터, 컨트롤 유닛, ALU 와 협력하여 연산이 이루어짐.

데이터 이동의 기반 버스 시스템

  • 데이터 버스
    • 데이터 이동
  • 어드레스 버스
    • 주소 이동
  • 컨트롤 버스
    • 컨트롤 신호 이동
