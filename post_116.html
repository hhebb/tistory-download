계수 정렬 (counting sort)

  • 개념

    입력의 최대값만큼 배열을 만들어 그 배열의 인덱스 번호를 원소로 변환하는 아이디어.

  • 비고
    인덱스가 곧 원소라는 핵심 아이디어로 다른 상황에서도 쓰일 수 있다.
    대소 관계를 비교하지 않는 비 비교 정렬? 이다.

    배열을 미리 만들어야하기 때문에 최대 입력값이 적당히 작아야한다.(많아도 천만 따리 까지 안가는게 좋을 듯)
    대신 입력의 갯수는 얼마든지 커버칠 수 있다.
    정렬과정이 필요없이 입출력만 해주면 되기 때문에 $O(N)$ 시간복잡도로 결과를 낼 수 있다.

def counting_sort(arr):
    max_val = max(arr)
    result = [0] * max_val
    for num in arr:
        result[num-1] += 1

    for i in range(len(result)):
        for j in range(result[i]):
            print(i,end=' ')

arr = random.sample(range(100), 20)
print(arr)
counting_sort(arr)

기수 정렬 (radix sort)

  • 개념
    자릿수를 차례대로 확인하며 정렬을 한다.

  • 비고
    자릿수가 있는 정수, 문자열을 정렬할 수 있다.
    물론 역순으로 정렬이 가능하고, 사전순 등으로 정렬할 수 있다.
    역시 대소 관계를 비교하지 않으며, 기수 정렬은 안정 정렬 이다.
    시간 복잡도는 최대 자릿수 k, 입력 크기 n 의 곱, $O(KN)$ 에 불과하다.

def radix_sort(arr):
    bucket = [[] for i in range(10)]
    max_order = len(str(max(arr)))
    tmp_result = arr[:]
    for order in range(max_order):
        bucket = [[] for i in range(10)]
        tmp_result = []
        for i in range(len(arr)):
            str_num = str(arr[i])
            if len(str_num) <= order:
                bucket[0].append(arr[i])
            else:
                bucket[int(str_num[-order-1])].append(arr[i])

        for b in bucket:
            for num in b:
                tmp_result.append(num)

        arr = tmp_result[:]
        #print(bucket)
        #print(tmp_result)
    print(arr)
arr = [1, 10000, 100, 43, 91, 3203, 8300]
radix_sort(arr)
