개념

  • 아주 펀더멘탈한 내용인데 요즘 간과하기 쉬운 개념.
  • 왜 컴파일하면 빠르고 아니면 느린지.

정적 타입 언어

  • 변수의 타입을 지정하여 공간을 미리 확보해놓는다.
  • 그리고 변수의 타입은 변하지 않고 충실히 동작한다.
  • 기계어로 컴파일을 해놓을 수 있다.
  • 기계어로 변환된 코드는 당연히 빨리 동작할 수 밖에 없다.
  • C 계열 언어, JAVA 등

동적 타입 언어

  • 변수의 타입이 동적으로 변한다.
  • 런타임에 계속 크기가 변하기 때문에 컴파일을 할 수 없다.
  • 컴파일이 안 하면 코드를 바로 실행하여 변환된 기계어보다 느리게 동작할 수 밖에 없다.
  • python, javaScipt, ruby 등이 있다.

비고

  • 동적 타입 언어가 동적임에도 불구하고 빠른 속도를 요구하자 새로운 방법이 생김
  • 이종 언어 간 인터페이스 등.
  • 파이썬에선 ctypes 모듈로 c 언어로 작성된 dll 등 기계어로 번역된 코드를 갖다 쓸 수 있다.
  • 동적 라이브러리는 윈도우에서 dll, 리눅스에서 so 파일인데 OS 커널 영역에 접근하여 사용하는 듯 하다.
