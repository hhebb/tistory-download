트리를 구성헐 수 있는 구현력을 길러야 한다.

트리 순회

  • 난이도 - 하
  • 유형 - 트리, 구현, 트리 탐색
  • 핵심
    트리를 구성할 줄 알아야 함.
    기본적으로 node 클래스를 만들어야 하고, 이 문제의 경우엔 딕셔너리로 트리 정보를 관리할 수 있다.
    트리 전위순회 중위순회 후위순회
    루트를 기준으로 루트 1 순위는 전위, 중간이면 중위 마지막 순위이면 후위
  • 비고
    트리 구성하는 프로그램 조차 작성하지 못했다.
    입력 형식이 조금이라도 바뀌면 정신을 못차린다.
    이 문제 형태에는 딕셔너리로 트리를 구성할 수 있다.
    그냥 트리에 대해 익숙하지 않았기 때문에 어렵게 느껴졌다.
    어떻게 구성해도 논리적으로 구조가 된다면 그 자료구조를 구현한 것이 된다.
    문제의 목적에 맞게 기능을 수행할 수 있으면 어떤 형태든지 가능하다.
    머리에 자극이 온다.
# 트리 순회
class Node:
    def __init__(self, data, left=None, right=None):
        self.data = data
        self.left = left
        self.right = right

def prev(node):
    print(node.data,end='')
    if node.left != '.':
        prev(tree[node.left])
    if node.right != '.':
        prev(tree[node.right])

def mid(node):
    if node.left != '.':
        mid(tree[node.left])
    print(node.data,end='')
    if node.right != '.':
        mid(tree[node.right])

def post(node):
    if node.left != '.':
        post(tree[node.left])
    if node.right != '.':
        post(tree[node.right])
    print(node.data,end='')

n = int(input())
tree = {}
for _ in range(n):
    data, left, right = input().split()
    tree[data] = Node(data, left, right)

prev(tree['A'])
print()
mid(tree['A'])
print()
post(tree['A'])

트리의 높이와 너비

  • 난이도 - 중

  • 유형 - 트리, 구현

  • 핵심
    중위 순회의 특징을 이용한다. - x 축 기준으로 왼쪽부터 차례로 순회를 할 수 있다.
    레벨 값을 함께 저장할 수 있어야 한다.
    노드 클래스에 다양한 정보를 넣을 수 있어야 한다.

  • 비고
    중위순회하면서 레벨 값을 함꼐 기록하던가, 순회 한 다음 레벨을 기록하던가
    한꺼번에 하던가 나누던가 둘 중 아무렇게 해도 됨.
    나누면 코드가 길어질 수 있다. 편한대로 하면 될 듯하다.
    이 문제는 합치는게 효율적인 듯함.
    구현력이 뒷받침되어야 원활하게 풀 수 있다.
