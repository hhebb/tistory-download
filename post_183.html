내장 객체 계속

String Constructor

  • new String() 으로 생성. 인자는 전부 문자열로 변환된다.
  • new 안쓰면 인자의 리터럴을 반환한다.

String 프로퍼티

  • String.length - 길이를 반환

String 메서드

  • 항상 새로운 문자열을 만들어서 반환한다. 원시 값이므로!
  • String.prototype.charAt(n) - 해당 인텍스 문자 반환
  • String.prototype.concat(s1, s2) - 문자열 합치기. 이것보다 + 연산이 성능에 더 유리.
  • String.prototype.indexOf(str) - str 이 처음 발견된 인덱스 반환
  • String.prototype.lastIndexOf(str) - str 이 마지막에 발견된 인덱스 반환
  • String.prototype.replace(searchVal, change) - 검색한 모든 문자열을 교체한다.
  • String.prototype.split(sep) - 쪼갠것을 반환. 기존의 문자열은 유지.
  • String.prototype.slice(s, e) - 슬라이싱 결과 반환

배열 리터럴

  • 그냥 파이썬처럼 [] 쓴다.
  • 배열도 객체다.

배열 추가 삭제

  • 빈 배열에 아이템 추가할 때, 파이썬의 딕셔너리처럼 추가한다.
  • 충격적인것은 인덱스를 지정해서 값을 지정할 수 있다?
  • 삭제는 delete 연산자 사용.
  • 충격적인 것은 delete 는 값만 삭제되고 길이는 안 변한다.
  • splice 를 사용해야 길이도 바뀐다...

배열 순회

  • for, for of, foEach 를 사용한다.

배열 프로퍼티

  • Array.length

배열 메서드

  • Array.from() - 이터러블 객체로 새로운 배열을 생성.
  • Array.of(,,,) - 전달된 인수를 요소로 배열을 만들어 반환
  • Array.prototype.indexOf(n) - 검색 후 인덱스 반환
  • Array.prototype.push(i) - 마지막 위치에 아이템을 추가. 원본을 수정한다.
  • Array.prototype.pop() - 마지막 원소를 삭제하고 반환. 원본을 수정한다.
  • Array.prototype.shift(i) - 첫번째 위치에 아이템을 추가. 원본을 수정한다.
  • Array.prototype.unshift() - 첫번째 원소를 삭제하고 반환. 원본을 수정한다.
