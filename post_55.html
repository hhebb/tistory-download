난이도

프로그래머스 level 2, 카카오 2018 블라인드 채용 문제

유형

시뮬레이션
빡구현

특이사항

블럭의 움직임을 종이에 그려서 시뮬레이션 해야함.
위아래로 pop 하고 땡겨지므로 행렬을 뒤집은 형태로 만들어야 함.
2 차원 배열의 형태를 미리 고려한 후에 작성해야 함.
시뮬레이션이므로 여러 단계로 로직이 나뉘어져 있다.

총평

그다지 어렵지는 않은 문제였음.
미리 시뮬레이션을 하고 로직의 동작을 더 길게 따라가보았다면 중간에 로직을 수정하는 일은 없었을 것.
핵심은 배열을 뒤집어서 시작하는 건데, 실수를 하더라도 약간의 수정만 하면 된다는 것을 캐치하면 좌절하지 않을 수 있음.
그러나 실제 테스트에선 시간내에 못풀었을 것임.
다른 문제들처럼 지구력이 딸려서 힘들었음.

def solution(m, n, board):
    answer = 0

    matrix = [['-'] * m for i in range(n)]
    for ir, line in enumerate(board):
        for ic, f in enumerate(line):
            matrix[ic][m-ir-1] = board[ir][ic]

    check = [[False] * m for i in range(n)]
    friend = None
    to_remove = int(1e9)

    while to_remove > 0:
        check = [[False] * m for i in range(n)]
        to_remove = 0

        for c in range(m - 1):
            for r in range(n - 1):
                friends = matrix[r][c]
                # 4 블럭 만족 - 체크 해준다.
                if matrix[r][c] != '-' and matrix[r][c] != '_':
                    if (matrix[r + 1][c] == friends) and (matrix[r][c + 1] == friends) and (matrix[r + 1][c + 1] == friends):
                        check[r][c] = True
                        check[r + 1][c] = True
                        check[r][c + 1] = True
                        check[r + 1][c + 1] = True
                    else:
                        continue

        # 제거할 블럭 갯수, 제거할 블럭 플래그 설정
        for ir, r in enumerate(check):
            for ic, c in enumerate(r):
                if check[ir][ic]:
                    matrix[ir][ic] = '-'
                    to_remove += 1

        answer += to_remove
        if to_remove == 0:
            break

        for r in range(m):
            pos = 0
            removed = 0
            while pos < len(matrix[r]):
                if matrix[r][pos] == '-':
                    matrix[r].remove('-')
                    removed += 1
                else:
                    pos += 1

            for remove in range(removed):
                matrix[r].append('_')

    return answer
