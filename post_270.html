[2 강] - 윈도우에서 유니코드

문자셋의 종류와 특성

  • 문자열 함수들의 경우 SBCS 기반인 경우가 많지만 유니코드를 쓰면 문제가 발생
  • 한 가지 방법을 통일해야 한다!
  1. SBCS - Single Byte Character Set
    • 문자를 표현하는데 1 하이트 사용
    • 아스키 코드가 포함된다.
  2. MBCS - Multi
    • 한글은 2 바이트, 영문은 1 바이트 사용
    • 경우에 따라서 표현하는 방식이 Multi
  3. WBCS -Wide
    • 문자를 표현하는데 2 바이트 사용
    • 유니코드

c 에서 한글

  • sizeof 는 메모리 크기
  • strlen 은 문자열 크기
  • 한글은 문자열 크기에 문제가 된다.

WBCS 기반의 프로그래밍

  • SBCS 를 위한 두가지
    • char 대신 wchar_t 사용 - 문자를 2 바이트로 표현
    • "ABC" 대신 L"ABC" 사용 - 유니코드로 문자열을 표현
  • WBCS 기반 문자열 선언 예
    • wchar_t str[] = L"ABC";

main 함수 유니코드화

  • main 함수를 wmain 로 변경.
  • 매개변수를 유니코드 문자로 받는다.

MBCS, WBCS 동시 지원

  • window.h 에 다 정의되어 있음.
  • 호환성을 위한 프로그래밍
  • 매크로로 문자열 타입 지정을 사용하여 일관되게 사용.
  • unicode 관련된 타입도 매크로로 정의되어 있음.
  • 가급적 매크로를 지정하고 함수 및 타입을 치환하도록 프로그래밍 하기.
