오랜만에 기록

매일 공부한 거 블로그에 대략 정리만 하다가 오랜만에 이렇게 기록을 해본다.
사실, 강의를 하나하나 끝낼 때마다 머리가 어지러워져서 체크포인트를 찍어두고 싶어서 가끔 이런다.
특히 요즘은 더 많은 양의 지식을 받아들이느라 살짝 부하가 걸린다.
오늘은 요즘 나의 메인인 웹, JS 공부에 대한 느낀점을 풀어본다.

갑자기 웹 공부?

한 두 달 전에 그러니까 추석 전후로 해서 나 자신의 상태를 객관적을 바라보고 진로를 정하는 과정에서 웹 개발은 어떨까 라는 생각을 했다.
그동안 웹이라는 것에 엄청난 오해를 갖고 있던 터라, 그 사건은 정말 나에겐 큰 변화였다.
특수한 상황에서 현실적으로 몇가지 이유를 들어가며 웹 개발을 해야하는 이유에 대해 논리적으로 따져본 다음 그 날 이후 닥치고 웹을 공부하기 시작했다.
결론적으로 난 현재 웹이 IT 기술의 fundamental 이라는 생각과 가깝다.
파면 팔수록 신기하고 재밌고 또 많은 기회가 있으며, 기술의 집약체라고 생각된다.
취업을 위해서 웹을 선택한 부분도 있지만 웹은 충분히 익힐 필요가 있다고 생각하여 한동안은 계속 공부할 생각이다.

공부 과정

웹은 개발자의 수요가 많은만큼 공급도 많다.
그래서 좋은 강의와 자료가 많다.
웹은 완전 쌩판 처음이라 HTML, CSS 먼저 죠지기로 했다.
문제는 자료가 너무 많아서 좋은 것을 찾기가 어려웠다.
얼마간의 서칭을 통해 텍스트로 이루어진 학습자료를 접하고 html, css 를 훑었다.
양이 적지는 않았다.
난 마음이 급해서 실습은 건너뛰고 개념 정리만 쭉 했다.
그리고 바로 js 를 공부하려고 했다.
사실 백엔드 하려고 웹을 공부한 것이었다.
그래서 프레임워크를 정하고 공부하려고 했다.
하지만 어려움을 느껴 다시 JS 로 돌아와서 JS 먼저 익혔다.
지금은 JS 문법과 예제를 따라한 상태이며 익숙해지려고 하는 단계다.

어려웠던 점

먼저 웹이라는 개념이 머릿속에서 잘 안 그려졌다.
서버와 클라이언트는 수도 없이 들었지만 움직이는 과정은 도저히 매끄럽게 이해가 되지 않았다.
마치 속을 들여다 볼 수 없는 블랙박스 같은 느낌이었다.
지식을 계속 접하다보니 이해가 되기 시작했지만 그러기까지 매일매일 생각을 했다.
특히 며칠 전 접한 유튜브 얄팍한 코딩사전님의 웹 시리즈는 답답한 속을 뻥 뚫어주었다.
이 때 한 번 특이점이 온 것 같았다.

JS 공부는 정말 머리가 아팠는데, 일단 자바라는 단어가 들어감에 거부감이 생겼다.
웹을 하려면 어쩔 수 없이 JS 를 해야해서 껄끄러운 상태로 공부를 시작했다.
정말 자료가 많았고 어느정도 수준을 선택해야는지도 잘 몰랐다.
언어가 다 비슷비슷하지라는 생각으로 js 문법이 녹아든 nodejs 강의를 듣다가 도저히 못참겠어서 도중에 바로 껐다.
그 후로 텍스트로 길게 정리된 poiemaweb 라는 사이트에서 JS 를 제대로 시작했다.
퀄리티는 정말 좋았는데 하면 할수록 JS 가 점점 더 싫어졌다.
일단 설계상 오류가 많은 언어이고 코딩 스타일도 제각각인것 같았고 심지어 ES6+ 라는 대규모 변화로 인한 개념의 확장 등이 원인이었다.
때문에 다른 언어를 익힐 때 고민 안했던 부분에 많은 시간을 할애하고 JS 만의 괴랄한 점을 인지하고 하는 등의 과정에 현기증이 났다.
어거지로 50 개 정도의 JS 문법을 정독하고 나니 역시 머리에 남는게 얼마 없었다.
중요한 개념인 스코프, 프로토타입, 렉시컬 등 몇개는 강렬해서 기억에 남지만 나머지는 거의 다 까먹고 그런게 있었지 라는 정도의 기억의 파편만 남았다.
물론 코드를 작성할 수는 없었다.
그래서 이 사이트는 참조용으로 두고 다른 실용적인 강의를 찾았다.
구름 edu 에 있는 js 초스피드 문법 강좌였는데 오늘 당일 결제하고 금방 다 들었다.
문법 기초 정도는 하루만에 다 듣고 익히겠다는 다짐을 했다.
이런저런 강의 듣는게 좀 별로였지만 생각해보면 파이썬 공부할 때도 이것저것 다 찾아가며 공부했었고 사실 JS 공부보다 더 비효율적을 했었다.
그리고 정말 한정적으로 언어를 사용해서 잘못 알고있거나 모르는 개념이 더욱 많았다.
따지고보면 파이썬은 정말 최근에야 조금 잘 쓰게 됬는데, 코테 준비하면서 다양한 메서드를 쓰면서 가능한 것이었다.
어쨌든 JS 를 공부하면서 정말 껄끄러웠지만 빠른 시간안에 뭔가 이해를 많이 한것 같다.
특히 JS, 웹 공부를 하면서 웹에서 두드러지는 개념들에 대해서 많이 알았다.
바로 비동기, 콜백이다.
예전에 비동기라는 단어는 많이 들었지만 도저히 쓸 일이 없어서 바로 까먹고 말았지만, 웹에서는 정말 중요하다는 것을 알게되니 머리에 쏙쏙 들어왔다.
일찍이 파이썬에도 비동기가 있었지만 알 필요가 없었다.
JS 에서는 정말 중요하다는 것을 알게되었다.

현재 상태

지금은 웹에 대한 큰그림이 그려지는 상태다.
솔직히 며칠전까지만 해도 그냥 무작정 해나가는 처지였다.
내가 정말 IT 머리가 없는건가? 라는 생각을 하며 기운이 빠져있었다.
IT 기술은 생각보다 오랜 시간동안 그리고 엄청난 속도로 발전해서 그 양이 방대하다.
나도 IT 관련 일을 조금 했고 코딩도 좀 했지만 영역이 다르면 영 따라가기가 힘들다.
그래도 다행히 지금은 웹 전반적인 개념을 잡았다고 생각이 될만큼 든든한 기분이다.
JS 도 어느정도 핵심정리가 되었고 뭐 연습은 더하면 되겠지.

이제 조심스럽게 뭔가를 만들어 볼만하다고 생각이 된다.
틈틈히 서비스, 포트폴리오 구상한것들을 정리해놨지만 그래도 연습을 한 두 번 한 다음 나만의 서비스를 만들려고 한다.
무료 클론코딩 강의를 한 두개 듣고 node, express 에 익숙해지고 진행해야할 것 같다.
사실 JS 공부는 node, express 강의를 위해 선수학습한 것이다.

계획

지금이 11 월 중순이 되어가니까 11 월 끝날때까지 2 - 3 개의 클론코딩 강의를 듣고 감을 익힐 것이다.
그리고 12 월 한 달 동안은 순수 내 힘으로 서비스 2 개, 그리고 나만의 정적 블로그 하나를 파서 포트폴리오를 2 - 3 개 가량 만들 생각이다.
그 후에는 꾸준히 계획을 업데이트 해나갈 것이다.
