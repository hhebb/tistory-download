html, css 연습

js 는 문법 학습 단계이므로 일단 문서 구조와 렌더링에 대한 연습이 필요하다.
이는 연습이나 클론코딩이 직빵이지만 그 전에 개념과 요소들을 정리할 필요가 있다
아직 익숙하지 않은 초심자의 입장에서 전체적인 내용을 떠올릴 수 있도록 정리하는 목적이다.

html, css 정리

html

html 은 문서의 구조를 정의한다.
태그로 문서를 표현하는데 페이지를 어떻게 구성할지 생각을 해야 한다.
중요한 것은 시맨틱 웹 기준을 따라야 한다.
큼직한 것부터 나누고 그 안에 계층을 나누어야 한다.

meta 태그

중요한 기타 정보들을 모두 정의한다.
charset

레이아웃

div 와 함께 큰 틀을 잡는데 활용한다.
header footer aside nav section article 등이 있다.

form

서버로 전달할 데이터를 보내야 할 때, 인터랙션 요소가 필요할 때
form 태그 안에 input, button, textbox, select-option, field set 등

멀티미디어

img, audio, video 등이 있다.

텍스트, 리스트

p, pre, em, strong,

참조

a, link, script

속성들

id, class, title, href, target, src, type

css 정리

css

css 는 구조화된 html 문서를 어떻게 보여질지 정의하는 것이다.
html 에 비해서 다소 양이 많고 복잡하게 느껴질 수 있다.

셀럭터

가장 기본이 되는 selector 개념이다.
스타일을 줄 대상을 지정하는 것이다.
html 구조가 중첩이 많이 되어있으면 하나하나 지정할 수 있도록 다양한 셀렉터가 있다.
태그, id, class 등 기본적인 것
다중 선택, 후손, 자손, 형제 등 더 고급 선택자
그리고 가상 클래스, 가상 요소 등의 특수한 선택자가 있다.

레이아웃 요소

가장 중요한 것 중 하나인 레이아웃을 결정하는 것
position 요소, display 요소는 매우 중요한 개념이다.
position 은 relative, absolute, fixed, static.
display 는 inline, inline-block, block 등이 있는데 직접 쓰면서 익숙해져야 한다.
그리고 float 이 있는데 아주 잘 써야한다.
기존의 배치에서 벗어나는 형태인데 얘를 감싸는 요소에 후처리를 잘 해야한다.

박스 모델

blcok 요소, inline-block 요소일 때 여백, 테두리, 내용물 크기 등을 조절한다.
border, margin, padding 과 같은 외부 여백과 width, height 등의 내용물 크기가 있다.

텍스트

보통 inline 요소다.
폰트 크기, 색깔, 종류, 위치, 정렬 등이 있다.

정렬

inline 요소와 그렇지 않은것으로 나뉜다.
inline 요소는 대부분 텍스트 이므로 text align 을 쓰게 된다.
이외에는 박스 모델의 margin 을 이용하거나, 부모의 padding 값을 조절한다.

트랜지션

특수한 상황에서 요소의 변화를 나타낸다.
transition 요소에 변화시킬 요소와 조건 등을 작성한다.
이벤트 기반으로 일어나는데, 가상 클래스 선택자를 주로 사용하게 된다.
왠만하면 js 로 하는게 질적으로나 성능으로나 더 좋다.

애니메이션

특수한 상황이 아닌 스스로 요소가 움직이도록 하는 것.
키프레임을 정의하고 작성하게 된다.

트랜스폼

요소의 단순한 이동을 넘어서 복합이동, 회전, 비틀기, 스케일 등을 실현한다.
트랜지션이나 애니메이션과 합쳐져서 비쥬얼적으로 멋진 효과를 낼 수도 있다.

반응형

요즘 제일 중요한 것 중 하나이다.
html 에 viewport 태그를 추가해준다.
그리고 css 에 @미디어 쿼리를 이용해 해상도에 맞게 반응하도록 한다.
브레이크 포인트를 3 개 정도 지정하여 사이즈가 변할 때 마다 레이아웃이 변하도록 한다.
이는 역시 연습이 답이다.

사실 주로 사용되는 css 요소들은 정해져 있고, 자주 쓰다보면 익숙해진다.
그리고 요소들을 다 모르더라도 기본 원리와 핵심 기법들만 익히면 얼마든지 멋진 페이지를 만들 수 있다.
난 프론트엔드를 메인으로 하진 않을것이지만, 풀스택을 목표로 두고 공부한다는 점,
내가 만들고 싶은 서비스가 생길 때, 혹은 외주 일을 찾으려고 할 때를 위해서 일정 수준 이상으로 익히려고 한다.
웹 페이지를 만드는 것 자체만으로도 즐거운 일이니까 충분히 즐길 수 있다.
