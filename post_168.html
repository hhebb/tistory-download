네트워크 구성

  • 네트워크 엣지 - 호스트
  • 네트워크 코어 - 엣지 중심에 라우터.
  • 엣지와 라우터를 연결해주는 링크들.
  • 링크는 무선 링크(wifi, lte layer), 유선 링크.

네트워크 엣지

  • 클라이언트, end system
  • 클라이언트 서버 모델
  • peer to peer

인터넷 서비스

  • TCP
    reliable transfer - 신뢰성이 있는 통신 방식
    byte order - 바이트 순서대로 도달.
    flow control - 수신자의 속도를 맞춰서 통신하게 함.
  • UDP
    connectionless
    no flow control
    unreliable data transfer
    no congestion control
  • UDP 도 사용처가 있다.
    신뢰성이 엄청 중요하진 않을 때
    보이스 서비스 등.
    컴퓨팅 파워가 덜 든다.

프로토콜

  • 통신에 필요한 미리 정해진 규약

네트워크 코어

  • 라우터
  • 데이터 전송 방식
    circuit switching
    packet switching

circuit switching

  • 옛날 유선 전화망 전송 방식
  • 정해진 경로를 기준으로 데이터를 보냄
  • 유저들의 수가 제한된다.
  • 한정된 유저들이 통신을 안하는 상태에서도 망을 점유하고 있어 비효율적이다.

packet switching

  • 현재 인터넷의 전송 방식
  • 들어온 패킷을 기준으로 데이터를 보냄
  • 유저가 제한되지 않는다.
  • 일시적으로 데이터를 사용하므로 동시에 더 많은 사용자를 수용할 수 있다.
  • 하지만 유저가 몰리는 순간에는 오버헤드가 발생한다.

라우터에서 발생하는 전송 지연

  1. process delay
    데이터 체크, 전송 경로 정하는 데 생기는 지연시간
    라우터에서 발생
  2. queueing
    outgoing edge 로 나가는 데 대기열이 있을 때 대기하는 시간
    라우터의 임시 버퍼에서 기다릴 때
    라우터에서 발생
  3. transmission delay
    데이터의 처음부터 끝까지 라우터에서 링크로 나가는 데 걸리는 시간
    회선의 대역폭에 좌우 됨.
  4. propagation delay
    링크에 올라와서 edge 까지 가는 데 걸리는 시간
    광속.
    건드릴 수 없음.

전송 지연 방지

  • process delay, transmission
    장비를 좋은거로 바꾸면 해결.
  • propagation delay
    건드릴 수 없음.
    광속임
  • queueing delay
    가장 control 하기 어려움.
    유저가 몰리면 지연됨.
    queue 의 크기가 한정되어 있는데 유저가 몰리면 유실이 될 수도 있다.
    사실상 거의 대부분의 유실 원인이다.
    TCP 는 신뢰성을 보장하기 위해 패킷 유실 시 재전송한다.
    라우터가 아닌 유저가 다시 재전송한다.
    라우터의 역할은 고속 전송에 특화되어야 한다.

패킷

  • 통신 비트의 집합

패킷 스위칭

  • 패킷은 한 뭉치다.
  • 패킷의 모든 비트가 링크에 올려질 때까지 기다렸다가 (transmission) 한 번에 propagation 해야 한다.
