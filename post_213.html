기초 지식

노드

  • 서버사이드 JS
  • 구글 V8 엔진 이 위에서 돌아감.
  • 단일 스레드 이벤트 기반 비동기 입출력

비동기 처리

  • 여타 다른 서버 환경은 클라이언트 요청을 비동기 처리하기 위해 새로운 스레드에 할당함
  • 이로 인해 부하가 발생한다.
  • 노드는 단일 이벤트 루프 스레드를 이용해 이벤트를 받아들인다.
  • 이벤트 처리는 내부에 있는 스레드 풀에서 수행하고 응답을 건네 준다.

기타 특징

  • 클라이언트에서도 사용되는 JS 를 백엔드 개발에도 사용할 수 있어 풀스택이 가능
  • npm 이 지원하는 다양한 라이브러리를 사용할 수 있다.

Node 아키텍처

  • 이벤트 처리, 스레드 풀, 입출력 등은 고속처리를 위해 c 언어를 기반으로 함.
  • 구글의 V8 엔진을 토대로 한다.

모듈

  • 기본 모듈 / 확장 모듈
    기본 제공 모듈과 따로 설치해야하는 모듈
    HTTP 등은 기본 제공
    express, mongoose 는 확장 모듈
  • 일반 모듈 / 네이티브 모듈
    네이티브는 c, c++ 로 작성된 코드. 빠르다.
    js 로 작성된 모듈은 일반 모듈.
    .node 는 네이티브로 컴파일된 것임.
    .js 는 순수 js 모듈
  • 글로벌 모듈 / 로컬 모듈
    어디서든 접근할 수록 전역으로 설치하면 글로벌 모듈. -g 옵션으로 설치
    특정 프로젝트 한정해서 사용할 목적으로 설치하면 로컬 모듈

모듈 식별자

  • 상대적 식별자
    ./ 은 현재 위치라는 뜻이며 기본 모듈이 아니면 반드시 이렇게 require 해야함
  • 절대적 식별자
    모듈의 절대 경로를 작성하여 require
  • 최상위 레벨 식별자
    ./ 없이 모듈 이름만 작성한다.
    기본 모듈일 때 이렇게 require 가능

모듈 내보내기 / 포함하기

  • 전역객체의 exports 나 module.exports 로 js 파일을 내보내기 한다.
  • 그리고 require 에서 내려 받는다.
  • exports 는 객체이므로 객체처럼 내보내야 한다.

전역 객체

  • 코드 어디에서나 접근할 수 있는 전역 스코프를 가진 전역 객체가 존재한다.
  • 브라우저 엔진에서 돌아가는 js 는 window 가 전역 객체이다.
  • node 환경에서는 전역객체가 따로 존재한다.
  • console 객체, precess 객체, exports 객체 등이 있어서 import 없이 바로 사용가능.

기타 기본 모듈

  • OS
    프로세스가 돌아가는 머신의 os 정보를 가져올 수 있다.
  • Utility
    잡기능
  • File System

    주로 에러 처리.

    addListner 는 이벤트와 핸들러를 연결한다.
    emit 은 해당 이벤트를 발생시킨다.

npm

  • 막강한 라이브러리 저장소. 파이썬의 pip 와 대응.
  • 하지만 pip 보다 더 많은 역할을 한다. 프로젝트 관리, 모듈 관리
  • 레지스트리에 등록된 라이브러리를 설치할 수 있다.
  • npm list - 모듈 목록
  • npm update - 모듈 업데이트

package.json

  • 원래 모듈 배포 목적으로 있는 건데, 프로젝트 관리에도 사용됨.
  • 의존성 관리, 스크립트 실행 명령어 관리 등을 한다.
  • npm init - 패키지 json 파일 생성
  • npm install -save - json 파일에 모듈 추가하고 설치

기타 모듈

    node monitor

    웹 소켓 프로토콜을 사용할 수 있게 한다.
