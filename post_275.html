[7 장] - 프로세스간 통신 (IPC)

프로세스 통신

  • 2 개 이상의 독립된 프로세스간 데이터 및 신호를 주고 받는 것.
    • 정확하게는 메모리를 공유.
    • src 에서 dst 로 주고받는 흐름만 생각하면 안됨.
    • 주고 받는 것이 아닌 갖다놓고 가져가는 것도 포함.
    • 즉 메모리 공간을 공유하는 의미이다!
  • 2 개 이상의 프로세스는 독립된 프로그램?
    • 하나의 프로그램은 여러 개의 프로세스로 형성되는 경우도 가끔 있다.
    • 이럴 때는 IPC 가 필요하다.
  • 프로세스는 할당받은 메모리외에는 정말 접근이 불가능.
    • mutex 등 안전성을 위해 일부러 허용하지 않는 것임.
    • OS 가 막음.
  • 프로세스 간 통신이 필요할 때
    • OS 가 제공.

메일 슬롯

  • dst 프로세스에서 우체통을 만든다.
    • OS 가 직접 주소와 함께 만들어준다.
  • dst 프로세스는 함수를 통해 간접적으로 이 우체통에 접근 가능.
  • src 프로세스는 dst 의 주소를 확인하여 데이터를 전송한다.
  • 메일 슬롯이라는 메모리 공간을 src 와 dst 프로세스가 공유하는 아이디어.
    • 그런데 단방향의 성격이 있다.
    • src 는 보내기만 가능하고 dst 는 받기만 가능하다.
  • 특이한 것은 프로세스들이 메일슬롯 자체를 공유할 수 있다는 것.
    • src 는 한 번의 전송으로 여러개의 dst 에 데이터를 보낼 수 있다.
  • 실제로 파일을 만들어서 여러 프로세스가 참조함으로써 IPC 의 개념을 구현할 수 있다.
    • 하지만 안전성의 문제가 있어서 그렇게 잘 안 쓴다.
  • 안전성을 추가한 방식이 메일 슬롯이다.
    • 어쩄든 파일 생성 기반으로 구현한다.
    • 대부분의 OS 가 메일 슬롯을 파일 시스템을 기반으로 구현한다.

Signaled vs Non-Signaled

  • 커널 오브젝트의 상태
    • 상태는 리소스의 현재 상황을 알리기 위한 것.
    • 프로세스 상태에 따라 signal 이 토글되는 것?
  • 상태는 signaled 여부로 나타낼 수 있다.
    • 물론 프로세스에 따라서 가지고 있는 상태가 다르다.

상태 관찰 시나리오

  • 프로세스가 생성되면 non signal 상태.
  • 프로세스가 종료되면 signaled 상태가 된다.

코어

  • 코어가 늘어나면 멀티 프로세싱으로 성능의 향상을 기대할 수 있다.
  • 부모 프로세스가 자식 프로세스의 결과를 기다려야 할 때까지 기다려야 한다.
    • signaled 상태를 감지하도록 한다.
    • 이 떄까지 부모 프로세스는 블락됨.
