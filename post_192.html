정적 웹, 동적 웹

개념을 알아보자.
출처 - 얄팍한 코딩사전
여기서 정적웹을 손님(클라이언트) 가 진열된 상품(리소스) 를 가져와 자기식대로 조리하는 컵라면,
동적웹을 요리사(서버)가 매번 조리(데이터 가공)해서 주는 음식이라고 비유함.
가공하는 것이, html 등의 문서도 해당하는 듯하다.

정적 웹

  • 변하지 않는 웹
  • 접속할 때마다 서버가 동일한 결과나 파일을 제공하는 웹
  • 서버에서 데이터를 가공해서 주는것이 아닌 이미 만들어져있는 코드, 리소스를 그대로 가져가 브라우저에서 돌아가는 서비스.
  • html, css, js, 리소스 등을 올려놓기만 하고 브라우저가 가져다가 알아서 쓰는 방식.
  • 종류
    소개 페이지
    댓글기능이 없는 블로그 등

동적 웹

  • 서버가 데이터베이스 등의 변화를 적용하여 서비스를 제공한다.
  • 접속할 때 마다 서버가 데이터를 가공하여 보여준다.
  • 웹 프레임워크로 만든다. php, node, jsp 등으로 만든다.

동적 웹이 항상 좋은것은 아님

  • 블로그 같은 경우 그냥 정적 웹으로 관리하는 것이 편함
