js 공부

웹 개발을 하는데 js 는 필수다.
쉬운 언어라고 생각했는데 지금까지 써왔던 C#, python 등의 언어와 차이점과 특징점이 꽤 있어서
문법과 특징에 대한 공부를 먼저 한다.

js 역사

  • js 는 처음에 웹 브라우저를 위해 만들어짐.

  • 파편화 극복과 표준을 위해 ECMA 표준을 만듬

  • 초창기에 ajax 등장함.

  • ES 6 는 클래스, 화살표 함수, let, const 등이 추가 되어 프로그래밍 언어의 틀을 갖춤

  • 구글이 js 엔진을 강화하여 js 의 중요성을 부각시킴

  • 2009 년에 node 가 나와서 브라우저에서만 국한된 js 를 서버에서도 사용가능하게 함.

  • 이외에도 여러 플랫폼이 나와서 앱, 데스크탑 등 응용 프로그램 개발에도 사용함.

ECMA

  • js 는 일반적으로 언어 측면의 ECMAScript + 클라이언트 web api(DOM, BOM, XMLHttp) 등을 모두 아우른다.
  • web api 는 ECMA 와 달리 W3C 에서 별도로 관리한다.

특징

  • js 는 인터프리터 언어이다.
  • 함수를 일급 객체로 취급한다.
  • 멀티 패러다임 언어이다 (명령형, 함수형, 프로토타입 기반)
  • 그리고 프로로타입 기반은 객체지향 언어이기도 하다.
  • 객체 지향 - 프로토타입, 클래스 기반

ES6

  • 대부분의 브라우저는 ES6 지원하지만 구버전 브라우저를 위해 babel 트랜스파일러 필요.
  • 또 import/export 지원안하는 브라우저를 위해서 webpack 모듈 번들러를 사용해야 한다.

node

  • node 는 주로 서버 사이드 개발에 사용
  • 개발에 필요한 빌트인 api 를 제공
  • 데이터를 실시간 처리하여 빈번한 I/O 발생하는 SPA 에 적합하지만 CPU 사용률이 높은 앱에는 별로임

웹 동작 과정

  • 프론트 개발에도 node 를 많이 쓴다.
  • 브라우저의 핵심 기능은 서버에 요청, 서버로부터 응답을 받아 화면에 표시하는 것.
  • html, css 는 각각의 파서로부터 파싱되어 DOM, CSSOM 트리로 변환되고 렌더 트리로 결합된다.
  • 이 렌더 트리를 기반으로 브라우저는 웹페이지를 표시한다.
  • js 는 렌더링 엔진이 아닌 js 엔진이 처리한다.
  • html 파서는 script 태그를 만나면 js 코드를 실행하기 위해 dom 생성 프로세스를 중지하고 js 엔진으로 제어를 넘긴다.
  • 이처럼 브라우저는 동기적으로 문서들을 처리한다.
  • 그래서 js 로딩에 의해 DOM 생성이 지연될 수 있다.
  • 그래서 js 는 body 태그 가장 아래에 위치시킨다.
  • dom 이 완성되지 않은 상태에서 js 가 dom 을 조작하면 에러날 수 있음.
  • html 요소들이 스크립트 로딩 지연으로 렌더링이 늦어질 수 있다.
