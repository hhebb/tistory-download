용어

셋 다 웹에서 사용되는 app 계층에서의 프로토콜이다.
그러니까 TCP 소켓을 기반으로 하는 프로토콜이라서 TCP 소켓을 가지고 있어야 한다.
웹 서버와 클라이언트가 통신하는 방식의 차이이다.

HTTP

  • 기존 naive http 는 요청마다 페이지를 새로 고침하거나 이동한다.
  • 페이지 전체를 다시 보내고 받아서 렌더링한다.

AJAX

  • ajax 는 xml request 를 이용하여 http dom 을 수정만 한다.
  • 단순 http 보다 더 효율적이고 빠르다.
  • 페이지 전체가 아닌 부분만 변하게 된다.
  • 어쨌든 클라이언트가 서버에 요청을 해야한다.

WebSocket

  • 웹소켓은 클라이언트 요청이 없어도 통신한다. 즉 양방향 소통이다.
  • 실시간으로 소통한다.
  • 서버가 보내는 데이터를 즉각 반영한다.

TCP

  • 결국 모두 TCP 소켓을 이용해야 하는데 이를 쌩으로 구현하려면 꽤 복잡해진다.
  • 그래서 HTTP, 웹소켓 등은 라이브러리로 지원된다.
  • 이들 내부에는 TCP 를 이용하여 각 상위 계층 프로토콜에 맞게 작성되었을 뿐이다.
