node 기본 개념

  • js 는 원래 브라우저에서만 돌아갔음
  • 이것을 프로그램처럼 밖에서도 쓸 수 있게 한 것을 nodejs
  • nodejs 는 동시에 웹 관련 프로그램을 돌려주는 런타임 환경이다.
  • 즉, js 실행파일을 만들어서 돌려준다.
  • 자바의 가상머신에 해당하는 듯
  • js 로 작성된 웹 앱을 돌려준다.
  • node 자체는 그저 하나의 환경일 뿐이고 node 만으로 웹 개발을 하기엔 바닥부터 짜야한다.
  • express 는 nodejs 를 위한 웹 프레임워크. node 안에 기본적으로 설치가 된다.
  • 헷갈리는 것은, nodejs 만으로 서버가 구동이 가능하긴 한데, node 를 웹서버라고는 하지 않는다.
  • 오히려 WAS 에 더 가깝다.
  • 아파치나 nginx 를 연동하여 쓸 수 있는데 그렇게는 많이 안하는 듯 하다..

기본

console.log() 는 아무데서나 쓸수가 있다. 로그
json 은 js 에서 핵심적인 객체
console.dir() - 객체를 그대로 출력
__filename, __dirname 은 전역변수로 언제나 접근 가능하다.
process - 전역적인 객체.

모듈

라이브러리처럼 동작할 필요가 있을 때
메인 파일에서 불러온다.
모듈 파일에 exports 라는 전역 객체에 함수를 할당한다.
메인 파일에 require('모듈') 로 exports 객체를 로딩한다.
그리고 모듈.함수() 로 사용 가능.

모듈 2

module.exports 에 할당하면 함수가 아닌 객체를 직접 할당한다.
이걸 더 많이 쓴다.

노드 외장모듈 디펜던시

npm init 을 커맨드에서 입력하면 설정파일을 json 으로 만들어준다.
그리고 다른 디바이스에서 npm install 만 쳐주면 디펜던시 외장모듈을 자동으로 설치한다.
npm install _module --save 를 하면 설치할 때 설정파일에 모듈이름이 저장된다.

노드 내장모듈
