사용자와 그룹

리눅스는 멀티 유저 시스템
root 는 관리자, 슈퍼 유저. 모든 작업을 하는 권한이 있음.
모든 사용자는 하나 이상의 그룹에 속함
사용자는 etc/passwd 파일에 정의됨.
시스템이 사용하는 여러 유저도 있다.
사용자를 생성하면 자동으로 해당 사용자의 그룹에 포함시킨다.

/etc/group

사용자 그룹 정보

/etc/passwd

사용자 정보

/etc/shadow

유저의 암호가 있지만 암호화되어 있어서 평문을 알 수 없다.

사용자 추가

adduser
--uid - id 지정
--home - 홈 디렉 지정

사용자 비번 지정 및 변겨

passwd -[user]

사용자 속성 변경

usermod

사용자 삭제

userdel
-r - 사용자 홈 디렉까지 삭제

암호 변경

passwd 명령어

암호 주기적으로 변경

chage [user]

사용자 소속 출력

groups
모든 사용자는 주 그룹 하나, 보조 그룹을 둘 수 있다.

groupadd

그룹 생성

groupmod

그룹 속성 변경

groupdel

그룹 삭제

파일 및 디렉토리의 소유, 허가권

permission
10 자리 파일 속성
1 자리는 파일 유형 - l, -, d, b 등
9 자리는 허가권 - 소유자, 소유 그룹, 그외 사용자 rwx

파일 허가권 변경

chmod 777
u, g, o - 유저 심볼

+x - 실행권한 추가

파일 소유권 벼경

chown
chgrp
관리자만 사용할 수 있다.
보안을 위해

파일 실행(execute)

./[file]
디렉토리를 타고 바로 파일 이름 적어줘도 실행됨.(다른 명령어 없이 경로만 쳐야함.)
실행파일 형태를 잘 만들어야 잘 실행됨.

유저 변경

su [username]
su 만 적으면 사용자계정으로 변경. 혹은 su root

링크

원본 파일 = inode 블록 + 파일 데이터
하드 링크 - 원본 파일과 inode 블록을 공유
심볼릭/소프트 링크 - 숨겨진 별도의 inode 블록을 만들고 원본 파일 포인터는 원본 파일을 가리킨다.

하드 링크

ln [대상 파일] [링크 파일]

심볼릭 링크

ln -s
