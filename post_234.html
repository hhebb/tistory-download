저장된 영상 스트리밍

  • 서버에서 클라로 데이터 보낼 떄 네트워크 딜레이가 있다.
  • 문제는 이 딜레이가 일정하지 않는 (jitter) 현상이 나타난다.
  • 서버는 일정하게 데이터를 보낸다.
  • 클라이언트는 일정하지 못하게 받는다.
  • 그래서 서버가 데이터를 전송하자 마자 재생을 시키면 안됨.
  • 조금 기다렸다가 재생이 되는데, 이를 버퍼링이다.
  • 초기 대기 시간이 너무 길어지지 않도록 잘 조절하면 됨.
  • 클라이언트는 버퍼에 받은 데이터를 저장해놓고 playout 하면서 데이터를 소비한다.
  • 버퍼에 조금이라도 남아있으면 괜찮음.
  • UDP 는 sender 가 속도를 결정하고 TCP 는 네트워크가 속도를 결정한다.
  • 유툽같은 스트리밍 서비스는 DASH 라는 app 계층 기술을 사용한다.
    UDP, TCP 를 그대로 쓰기에는 너무 느리다.
    Dynamic Adaptive Streaming HTTP
    데이터를 여러 퀄리티로 여러개 만들어놓고 작은 chunk 로 나눈다.
    chunk 테이블을 만들어 놓고 url 을 맵핑시켜놓는다.
    네트워크 상황에 맞게 퀄리티를 조절한다.
  • CDN 을 사용해서 해결하여 혼잡을 해결
  • 넷플릭스는 자체서버를 매니페스트 관리만 한다.
    CDN 업체 계약하고 거기에서 데이터를 내보내준다.
