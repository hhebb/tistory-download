운영체제
운영체제가 관리하는 하드웨어들을 알아본다.
운영체제는 시스템 자원들을 효율적으로 관리하고 사용자에게 서비스를 제공하는 SW 이다.

하드웨어

  1. 프로세서
    연산장치. CPU, GPU, 응용 전용 처리장치 등
  2. 메모리
    주 기억장치, 보조 기억장치, 디스크
  3. 주변장치
    키보드/마우스, 모니터, 프린터, 랜선 등

프로세서

연산장치.
컴퓨터의 장치의 동작을 제어한다.

레지스터

CPU 프로세서 내부의 메모리.
컴퓨터에서 가장 빠른 메모리
용도에 따른 분류 - 전용, 범용
변경 가능여부에 따른 분류 - 가시, 불가시 레지스터
저장 정보종류에 따른 분류 - 데이터, 주소, 상태 레지스터
가시 레지스터는 c 언어에서 확인 가능
불가시 레지스터는 프로그램 카운터, 명령어 레지스터, 누산기 등이 있다.

운영체제와 프로세서

운영체제는 프로세서에게 처리할 작업을 할당 및 관리하는 역할도 함
프로그램의 프로세서 사용을 제어한다.
시간 조율, 사용시간 관리 등

메모리

데이터를 저장하는 장치. 기억장치
프로그램, 사용자 데이터 등을 저장.
종류 - 레지스터, 캐시, 메인 메모리(RAM), 보조기억장치(HDD, SDD)
레지스터가 가장 빠르고 보조기억장치가 젤 느리다

주 기억장치

프로세서가 수행할 프로그램과 데이터를 저장한다.
DRAM 을 주로 사용한다.
디스크 입출력 속도가 너무 느려서 생긴 프로세서와 디스크 사이의 저장장치이다.
용량이 크고 저렴하다.

캐시

레지스터와 마찬가지로 CPU 안에 내장된 메모리
L1, L2 등의 캐시 메모리
레지스터보다는 조금 멀리 떨어져있고 느리다.
CPU 로 향한 RAM 의 입출력 병목현상을 해결하기 위해 만들어짐

캐시 동작

보통 hw 가 알아서 관리한다.
캐시 힛 - 필요한 데이터 블럭이 캐시에 존재할 때 바로 사용
캐시 미스 - 데이터 블럭이 없을 때 RAM 에 접근해서 업데이트

캐시에 대하여 - 지역성

공간적 지역성 - 참조한 주소와 인접한 주소를 참조하는 특성(순차적 프로그램)
시간적 지영성 - 한 번 참조한 주소를 곧 다시 참조하는 특성(반복문)
프로그램은 모두 지역성을 띈다.
지역성은 캐시 적중률과 밀접하다.
캐시가 용량이 작지만 충분히 RAM 의 병목현상을 해소할 수 있다.

지역성

2 차원 배열에서 반복문으로 인덱스 참조를 할 때 캐시에 대한 이해가 있으면
더 효율적으로 짤 수 있다.
배열을 캐시 블럭 단위로 불러오므로 지역성을 고려하여 프로그래밍 할 수 있다.

보조기억 장치

HDD, SDD, SD 카드 등
데이터를 저장한다.
프로세서를 직접 접근할 수 없다. - 주변 장치라고 한다.
쓰려면 주기억장치를 거쳐야 한다.
데이터나 프로그램이 RAM 보다 크면 가상 메모리를 이용한다.
가상 메모리는 하드를 주기억장치처럼 쓰는것.

시스템 버스

하드웨어들이 데이터 및 신호를 주고 받는 물리적인 통로.
프로세서, 메모리, 저장장치 등이 통신하는 통로.
데이터 버스, 주소 버스, 제어 버스 등이 있다.
각각의 버스에서 정류장처럼 데이터나 주소들이 내렸다 탔다가 한다.

주변 장치

프로세서, 메모리를 제외한 하드웨어들
입력장치, 출력장치, 저장장치 등이 있다.

주변장치와 OS

장치 드라이버 - 주변 장치 사용을 위한 인터페이스
OS 는 드라이버를 통해서 장치를 제어할 수 있고 드라이버를 관리한다.
인터럽트 처리
파일 및 디스크
