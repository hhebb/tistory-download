웹 의 이해

인터넷

하나의 컴터는 여러개의 서버가 동작할 수 있다.
각각의 서버는 포트로 동작된다.
웹은 80 번 등
인터넷은 네트워크의 네트워크

HTTP

웹 브라우저 관련기술과 함께 개발됨
인터넷 상에서 데이터를 주고 받기 위한 프로토콜
지금은 HTTP 2 까지 등장함.

HTTP 작동방식

서버 클라이언트 모델을 따른다.
클라이언트가 서버에 요청을 보내고 서버가 클라이언트에게 응답을 한다.
무상태 프로토콜이라고도 한다.

  • 장점
    불특정 다수를 대상으로 하는 서비스에는 적합함.
    c-s 가 계속 연결된 상태가 아니라서 클라이언트와 서버간 최대 연결수보다 훨씬 많은 요청과 응답을 처리가능
  • 단점
    연결을 끊어버리는 무상태(stateless) 방식이라서 클라이언트의 이전상황을 알 수 없다.
    그래서 쿠키와 같은 기술이 등장함.

URL - uniform resource locator

인터넷 상의 자원의 위치
특정 웹 서버의 특정파일에 접근하기 위한 경로 혹은 주소
3 가지 부분으로 나뉨
접근 프로토콜 + 도메인 및 주소(IP) + 자원(파일)의 이름
머신에는 여러개의 sw 서버가 있는데 서버 당 하나의 포트를 할당받는다.

동작 방식

data-origin-width="0" data-origin-height="0" data-ke-mobilestyle="widthContent"

 invalid-file

  1. 클라이언트 --> 서버 연결
  2. 클라이언트 --> 서버 요청
  3. 서버 --> 클라이언트 응답
  4. 연결 종료
    요청할 땐 메세지 형식에 맞춰서(헤더+바디)
  • 요청 헤더 = 요청 method + 요청 URI + HTTP 프로토콜 버전
    get 메서드는 바디가 없다.
  • 응답 헤더 = 응답 프로토콜 + 응답 코드 + 응답 메세지 + 날짜, 이름 등등

웹 서버

웹 서버 SW

서버는 sw 를 말하지만 웹 서버 sw 가 동작하는 컴퓨터를 지칭하기도 한다.
웹 서버의 가장 중요한 기능은 클라이언트가 요청하는 문서나 리소스를 전달하는 것임.
브라우저나 크롤러가 요청하는 리소스는 서버에 저장되어 있는 정적인 데이터(이미지, css 파일 등)이거나 동적인 결과(웹서버에서 동작되는 프로그램)가 될 수 있다.
