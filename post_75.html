이미지 포맷

이미지는 웹페이지에서 매우 중요하다.
효율적인 포맷을 사용하지 않으면 웹이 무거워진다.

jpeg(jpg)

손실 압축 방법.
원본파일을 저장하는 데 적합하지 않음.

gif

사진, 움짤 등을 저장하는 포맷
무손실 압축 방법.
하지만 색을 최대 256 개만 사용가능
그래서 색상이 많은 사진을 저장하는 데는 부적합함.
그러나 로고와 같은 적은 색을 저장할 때는 선명함.
웹에서 애니메이션을 재생할 수 있음.

png

png 8 은 gif 를 거의 대체함.
애니메이션을 제외하고 같음.
압축률이 더 좋기 때문에 애니메이션이 아니라면 png 8 을 권장.
png 24 는 모든 색을 다 저장함.
투명도를 부드럽게 표현함.
대신 용량이 큼

img 요소

이미지를 나타내는 요소
속성으로는 src, alt 가 있다.
src 는 이미지 소스의 주소.
alt 는 부연 설명. 이미지 로드가 안될 때 표시됨. 그리고 검색엔진이 속성을 알아봄.
설명이 필요없는 의미없이 꾸미는 이미지의 경우에 alt 는 비워둬도 된다.
단 alt 속성을 명시는 해야하는데, 이는 웹표준을 지키는 것이다.
툴팁은 title 속성을 사용한다.

img 요소의 기타 속성

width, height - 이미지의 너비, 높이를 지정하는 속성이다.
<img src="" alt="" width="" height=""/>
이 속성들이 없으면 이미지 원본 사이즈대로 표시가 된다.
최초에는 찌그러져있다가 이미지들이 로드되었을 때 해당 크기만큼 늘어난다.
물론 css 로 조절 가능하다?

목록 요소

문서에서 목록, 리스트를 나타내는 요소들이다.

ol(ordered list)

말그대로 순서가 있는 목록.
ol 요소 안에 li 요소를 넣어 항목으로 표현한다.
<ol> <li> 내용 </li> </ol> - li 앞이 숫자로 표시된다.
기본 표현은 숫자이지만 css 로 다른 형식으로 표현 가능
ol 과 li 는 블럭 요소이다.
하지만 ol 안에는 반드시 li 만 들어올 수 있다.
그리고 li 안에 다른 요소를 넣을 수는 있다.
ol 안에는 오로지 ol 만 중첩가능하다.
ol 의 start 속성은 항목의 시작 속성을 바꿀 수 있다.
<ol start="10"> <li> </li> </ol>
ol 의 reversed 속성은 역순으로 시작함.
<ol reversed="reversed"> <li> </li> </ol>
reversed 속성의 값은 반드시 reversed 만 가능하다.
이러한 속성을 불리언 속성이라고 하고 속성만 선언하면 동작하도록 할 수 있다.

ul(unordered list) 요소

순서가 없는 리스트이다.
ol 과 거의 유사하고 자식으로 li 만 담을 수 있다.
start, reversed 속성은 사용할 수 없다.
<ul> <li> </li> </ul>

dl(description list) 요소

개념과 정의로 이러어진 리스트다.
개념은 dt, 정의는 dd 요소로 작성한다.
dl 은 dt, dd 만 자식으로 넣을 수 있다.
<dl> <dt> 개념 </dt> <dd> 정의 </dd> </dl>
dt 는 인라인 요소, dd 는 블럭 요소
dt 와 dd 는 1:다 관계다.

목록 중복 사용

ol, dl, ul 들은 서로 중첩이 가능하다.
웹에서는 들여쓰기 안에 숫자 리스트 등 으로 표현된다.

의미없는 요소

이전까지의 요소들은 모두 의미가 있었다.
하지만 딱히 의미가 없이 사용하는 요소도 있다.
이들은 그룹을 묶거나 css 로 조절하기 위해 만든다.
가장 많이 사용하는 요소들이다.

div, span

div 는 블럭요소, span 은 인라인 요소
둘 다 자기자신을 중첩할 수 있다.
특히 div 는 레이아웃 잡는데 많이 씀.
보통 class 속성과 사용한다.
class 속성은 원하는 이름을 넣고 css 에서 선택자로 사용한다.
<span class=""> 내용 </span>
class 속성은 모든 요소에 사용이 가능한 속성이다.

class 와 id 속성

이 둘은 css 에서 스타일을 주는데 자주 사용하므로 헷갈리기 쉽다.
id 는 문서에서 딱 한번만 선언가능함.
class 는 여러번 선언 가능하다.
그래서 class 는 넓은 범위의 개념, id 는 세밀한 내용을 적는다.
또한 class 는 띄어쓰기를 통해 여러개의 값을 넣을 수 있다.
이는 css 작업을 간편하게 할 수 있도록 해준다.
