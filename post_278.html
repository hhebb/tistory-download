블록체인이란

  • 블록체인은 데이터를 저장하는 기술임. (일종의 DB)
  • 가상화폐에서는 원장을 저장하는 DB .
  • 이 블록들을 연결 리스트 형태로 연결하여 체인처럼 만듬.
  • 연결하는 방법은 조금 특이하게, 해시를 이용한다??

해시 함수

  • 해시 자체는 다양한 데이터를 모두 고정된 크기의 데이터로 맵핑하는 것임.
  • 해시 함수는 원본 값에서 해시 값을 도출해주는 함수
  • bitly 도 해시다.
  • 충돌은 일어날 수 있지만 sha 256 은 거의 안난다.
  • 실제로 sha-1 은 충돌이일어나고 이걸로 암호 깰 수 있다.

해시 함수 예

  • sha 256
    • 원본 값이 조금만 바뀌어도 해시값은 크게 달라진다.
  • keccak 256
    • 요즘은 keccak 해시가 더 많이 쓰임?

SHA 256

  • 해시 결과가 256 bit 로 나옴.
  • 256 bit = 32 byte, 1 byte 는 16 진수 2 자리수.
  • 그래서 64 개의 문자로 pure 하게 표현한다.

블록 연결

  • 연결 리스트 형태로 구현한다.
  • 뒤에 오는 node 가 이전 node 의 주소를 기억하는 연결 리스트가 있다.
  • 보통 앞의 node 가 뒤의 node 주소를 가리키지만 블록체인은 위의 방식을 따름.
  • 이러한 블록들의 리스트가 블록체인이다.

블록

  • 블록은 헤더와 바디로 구분됨.
  • 나중에 생성되는 블록은 앞의 블록의 해시값을 가리킨다.
  • 이 해시값은 중복되기가 어렵다.
  • 왜냐면 시퀀스 정보도 추가되므로.
  • 이 해시를 메모리상의 주소처럼 가리키고 있어서 해시 포인터라고 한다.

블록 높이

  • 초창기에 블록을 수직으로 쌓는것처럼 묘사함.
  • 첫 블럭이 흔들리면 모든게 무너진다고 수직으로 쌓아서 설명함.

블록 생성 주기

  • 블록체인은 일종의 DB 인데 블록을 만들고 싶을 때 만든다.
  • 하지만 보통 주기적으로 생성하고 연결한다.
  • 비트는 약 10 분, 이더는 약 15 초, 클레이튼은 1 초.
  • 실제로 블록에 편입되기까지 시간이 더 걸린다.
  • 클레이튼은 서비스 지향적이라서 빠른 생성 주기를 택함.

블록체인 네트워크

  • 블록체인은 P2P 네트워크
  • P2P 는 분산 노드간의 네트워크 프로토콜.
  • P2P 와 마찬가지로 블록체인은 데이터가 분산 복제되어 탈중앙화된 구조다.
  • 모든 노드들이 같은 데이터를 가지고 있고 그렇게 되도록 동기화한다.
  • 이 P2P 아키텍처와 잘 맞아서 블록체인은 P2P 로 구성되어 있다.
  • 블록체인 네트워크는 블록이라는 데이터를 주고 받게 된다.
