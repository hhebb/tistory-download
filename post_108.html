메타 요소

head 요소 안에 들어가는 meta 요소.

문자셋 메타 정보

charset
<meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />
글자 깨짐 현상이 발생하면 이 메타 정보와 페이지의 문자셋이 달라 생기는 문제일 수 있음.
국내에선 UTF-8, EUC=KR 을 많이 쓴다. UTF-8 이 더 권당됨.

description, keyword 메타 정보

<meta name="Description" content="내용" />
<meta name="Keywords" content="키워드1, 키워드2, 키워드3" />
설명과 키워드 메타 정보는 검색엔진과 관련이 있다.
이는 페이지 제목과 같이 표현, 공유하기 같은 기능에 활용이 된다.
키워드 정보는 검색엔진이 검색을 용이하게 하지만 악용 사례가 있어서 요새는 수집안한다는 의견이 있다.

author 관련 메타 정보

<meta name="author" content="이름"/>
페이지의 저자를 나타낸다.
큰 의미를 가지진 않는다.

우회, 새로고침 메타 정보

<meta name="refresh" content="10;url=http://~~"/>
숫자만큼 초를 기다리고 다음 페이지로 이동한다.
이는 접근성에서 좋지 않으므로 301 리디렉션을 사용한다.

IE 모드 메타 정보

<meta http-equiv="X-UA-Compatible" content="IE=edge" />
하위 버전의 IE 에 렌더링되도록 전처리한다.
되도록이면 엣지 버전으로 맞춰서 하도록한다.

뷰포트 메타 정보

<meta name="viewport" content="width=1200" />
모바일이나 태블릿 환경 때문에 사용한다.
모바일에서 자동으로 축소되면 크기가 부정확하게 될 수 있으니 이 메타 요소를 명시하는것이 좋음.
레이아웃이 꺠지지 않도록 한다.

form 요소 - form, input 태그

서버에 데이터를 보내는 GET, POST 방식을 알아본다.

form 요소

<form action="http://~~" method="get" />
action 소성은 폼을 전송할 URL 을 지정한다.
method 속성은 get, post 가 있다.
get 은 URL 로 값을 방식이다.
post 는 서버에 URL 이 아닌 숨겨져서 보내지는 방식

get/post

get 은 공개, post 는 비공개
get 은 서버에서 받아오는 검색/조회 등에 사용
post 는 서버에 올리는 글쓰기, 삭제 등 서버의 값을 변화시키는 작업에 사용.

input 요소

주로 폼 안에 들어가는 input 요소.
input 요소는 텍스트, 버튼, 등등 UI 요소들이 들어간다.
<input type="text" name="url" />
name 속성은 서버로 전송될 때 할당되는 변수 이름.
URL 이름에 변수=값이 붙어서 전송 됨.
<input type="submit" value="전송" />
이는 input 요소가 폼 전송 버튼 타입인 경우.

label 요소

어떤 값을 넣어야 하는지 라벨을 붙여주는 요소.
<label for="id"> 아이디입력 </lable>
<input type="text" id="id" />
웹 접근성 기준 상 폼 입력 요소가 있다면 만드시 해당하는 label 요소를 갖고 있어야 함.

form 요소 2

textarea, input 요소

textarea 요소

input 요소의 text 타입과 비슷함.
대신 textarea 는 여러줄 입력이 가능
cols 속성 - textarea 는 높이, 너비를 가진 글 박스이므로 속성을 필수로 선언해야함.
<textarea name="content" cols="40" rows="8"> </textarea>
css 로 높이, 너비는 수정할 수 있지만 html 에 필수로 명시해야 하는것이 좋음.
textaarea 안에는 pre 요소처럼 공백 등이 그대로 노출되고 html 태그도 그대로 노출됨.

input 요소

폼 입력을 하는 주요한 요소.
type 속성에 따라 여러 형태를 보여준다.

input 요소 타입

type="text" - 텍스트 타입 인풋

<input name="abc" type="text" value="test" />
가장 기본적인 형태로, 텍스트 입력 받는 형태.
value 속성의 값은 해당 input 의 기본 값으로 들어간다.
전송하면 input 의 value 가 전송됨.

암호타입 인풋

<input name="pw" type="password" />
노출되면 안 되는 텍스트를 입력 받는 타입

라디오 버튼 형태 인풋

<label for="radio-1"> <input type="radio" name="agreement" id="radio-1" value="1" /> 예 </lable>
<label for="radio-2"> <input type="radio" name="agreement" id="radio-2" value="0" /> 아니오 </lable>
라벨과 함께 사용한 예시
라디오 타입은 같은 name 속성 값을 가진 요소를 기준으로 선택한다.(여기서 agreement)
폼 전송 시 체크된 input 의 value 값을 전달한다.

버튼 형태 인풋

버튼 형태를 나타낸다.
type="submit" 과 유사하지만, 폼을 전송하지는 않는다.
버튼 인풋은 js 로 동작을 구현하는 용도로 많이 사용됨.
button 요소로 대체될 수 있다.

파일 업로드 인풋

파일을 업로드 할 수 있는 형태
업로드할 파일을 선택할 수 있고 폼 전송 시 업로드 된다.
<input type="file" name="pic" />

폼 전송 인풋

submit 타입은 이 요소가 속한 form 을 제출하는 역할을 한다.
value 속성이 버튼의 이름이 되고 button 태그로 대체될 수 있음.
<input type="submit" value="폼 제출" />

리셋 인풋

button, submit 타입과 비슷한 버튼 형태 타입.
form 안에서 폼 값들을 초기화하여 디폴트 값으로 바꿔놓는다.
버튼 형태이므로 button 요소로 대체할 수 있음
<input type="reset" value="초기화" />

이미지 형태 폼 전송 인풋

submit 이 버튼 형태 전송이라면 image 는 이미지 형태로 폼을 전송한다.
단순히 이미지 버튼을 구현하려면 기존 button 을 css 로 처리하는 것이 좋음.
<input type="image" src="img" alt="내용 전송" />

숨김 형태 인풋

아무것도 보이지 않는 형태.
폼에 정보를 전송할 때 사용자가 입력한 정보 외에 정보들을 보이지 않게 함께 전송할 때 사용
<input type="hidden" name="abc" value="something" />

input 요소의 속성들

type 속성외에 자주 사용되는 속성들.
대부분 속성이 값 없이 선언하는 것만으로 적용된.

disable 속성

모든 타입 공통 속성.
input 요소 뿐 아니라 모든 폼 요소들이 사용가능한 속성.
사용하면 해당 요소를 비활성화 시킨다.
<input type="text" value="nothing" disabled />

readonly 속성

text, textarea 요소에 사용된다.
텍스트를 읽기만 가능하도록 한다.
<input type="text" value="" readonly="readonly" />

size 속성

text 타입에 해당하는 인풋 요소에만 해당하는 속성
해당 요소의 너비를 지정한다.
<input type="text" size="5" />
css 로 처리 가능하다.

maxlength 속성

text 타입에만 사용 가능.
글자수 제한 속성 값
<input type="text" maxlength="10" />

checked 속성

checkbox, radio 타입에만 쓸 수 있다.
기본 값으로 선택되도록 보여줌.
<label for="radio-1"> <input type="radio" name="agree id="radio-1" value="1" /> 예 </label>
<label for="radio-1"> <input type="radio" name="agree id="radio-1" checked="checked" value="1" /> 예 </label>

select 요소, option 요소

select 요소는 사용자가 폼에서 사용할 수 있는 옵션의 목록을 보여주는 폼 요소
radio 인풋과 비스하지만, select 는 서택된 한 가지만 보여준다.
그리고 여러개 선택할 수도 있다.
select 요소는 한 개 이상의 optin 요소를 가지고 있다.
사용자는 option 을 선택하고 그 요소 값이 서버에 전송됨.
<select name="score" id="select-id">
<option value="a">A</option>
</select>

select 요소의 속성

다른 폼 요소와 마찬가지로 disabled, name 속성을 사용할 수 있다.
select 요소만의 속성이 몇가지 있음.

multiple 속성

기본적으로 option 을 하나만 선택할 수 있지만 여러개 선택할 수 있게 한다.
ctrl, shitf 등을 누르거나 드래그로 가능
<select multiple="multiple> </select>

size 속성

option 을 몇개 노출할것인지 정하는 속성.
기본적으로 노출 수는 1, multiple 속성이 있을 땐 4 개가 기본값.
<select size="3"> </select>

option 요소의 속성과 optgroup

내부에 다른 요소를 포함할 수 없고 텍스트만 포함할 수 있다.
<select> <option> </option> </select>

optgroup 은 option 요소들을 그룹화 해준다.

label 속성으로 그룹의 명칭을 지어준다.
<optgroup> <option> </option> </optgroup>

button 요소

input 요소의 submit, reset, button 타입 속성들은 버튼 형태였다.
모두 이 button 요소로 대체 가능하다.
<button type="submit"> submit </button>
등으로 요소 이름을 바꿔준다.
type 을 명시하지 않으면 기본값으로 reset 타입으로 동작함.
input 과 차이점은, input 은 스스로 닫는태그, button 은 열고 닫는 태그라는 점.
그래서 span, img 같은 인라인 요소를 안에 포함할 수 있다.
css 로 스타일을 줄 때 더 다양한 스타일이 가능해 진다.
<button type="button"> <인라인 요소> </> </button>

fieldset 요소와 legend 요소

fieldset 요소는 폼 요소들을 그룹핑, 그룹의 이름은 legent 요소로 지어진다.
스크립트에서 접근 시 보다 용이하게 만들어준다.
또 사용자 접근성도 높여 준다.
<fieldset>
<legend> 로그인 </legend>
</fieldset>
legent 요소는 fieldset 요소의 첫번쨰 자식으로 사용해야 하며 인라인 요소를 포함할 수 있음.
fieldset 요소는 자식으로 fieldset 요소를 중첩할 수도 있음.
