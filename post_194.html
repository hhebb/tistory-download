FE 프레임워크

출처 - 얄팍한 코딩사전
프론트도 프레임워크로 개발한다.

개념

  • 동적웹은 요리해서 주는 식당이고,
  • SPA 는 종업원(서버) 가 불판과 고기를 주면 손님(브라우저) 가 직접 구워먹는 고기집이라고 비유함.
  • 서버는 기존처럼 html, css, js 파일을 주는데, 이 js 는 바로 리액트나 뷰 같은 걸로 작성된 파일.
  • 이 js 파일은 데이터에 따라 html 페이지를 렌더링한다.
  • 동적 웹에서 서버가 하던 일을 브라우저가 하게한다는 것.
  • 즉, SPA 프레임워크로 작성된 js 코드는 브라우저에서 알아서 렌더링하게 하는 것이다!

등장 이유

  • 클라이언트의 사용자경험도 중요해짐에 따라 발전함.
  • 기존에는 ajax 등의 기술로 SPA 를 구현함.
    하지만 일일히 코딩해야함.
  • 모바일 시대를 맞이하고 모바일 환경에서도 PC 와 동일한 웹 서비스를 요구함.
  • 서버가 데이터만 보내줘도 환경에 맞게 브라우저가 알아서 잘 렌더링하면 생산성이 높아진다!
  • 즉 프론트와 백을 분리하게 됨.

종류

  • react
  • vue
  • angular

CSR, SSR

  • 클라이언트에서 다 렌더링되는 SPA 는 CSR(client side rendering) 이라고 한다.
  • 빠른 구동이 장점이지만, 초기 구동이 느리고 SEO 가 부족하다는 단점이 있다.
    단 구글은 js 까지 크롤링해서 큰 문제는 없다고 한다.
  • SSR server side rendering 이다.
  • 이는 단순한 동적웹과는 조금 다른데, SPA 를 서버에서 렌더링 해주는 것이다.
  • 이렇게 하면 SEO 도 되면서 초기 구동 속도도 빠르게 된다.
  • SPA 에서 SSR, CSR 을 섞어서 목적에 맞는 서비스를 만들 수 있다.
