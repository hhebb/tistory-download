프로세스 관리

job, process

  1. job 은 작업, 프로그램이라고 한다.
    실행할 프로그램과 데이터.
    컴퓨터 시스템에 실행 요청 전의 상태. 아직 디스크에 있음
  2. process 는 실행을 위해 시스템(커널)에 등록된 작업이다.
    시스템 성능을 향상시키기 위해 커널에 의해 관리된다.
    job -> process

프로세스의 정의

실행중인 프로그램
커널에 등록되고 커널의 관리하에 있는 작업
능동적인 개체
실행중인 프로그램

프로세스 종류

역할에 따라 - 시스템 프로세스, 사용자 프로세스
수행방법에 따라 - 독립 프로세스, 협력 프로세스

자원의 개념

커널의 관리 하에 프로세스에게 할당/반납되는 수동적 개체
hw resource - 프로세서, 메모리 ,디스크, 키보드 등
sw resource - 메세지, 시그널, 파일, sw 등

process control block (PCB)

os 가 프로세스 관리에 필요한 정보를 저장
프로세스 생성 시 커널에 생성된다.
커널이 관리하는 영역에 저장이 된다.

PCB 가 관리하는 정보

pid - 프로세스 고유 식별 번호
스케줄링 정보 - 프로세스 우선순위 등
프로세스 상태 - 자원 할당, 요청 정보 등
입출력 상태 정보
컨텍스트 저장 - 프로세스의 레지스터 상태를 저장 등
PCB 참조 및 갱신 속도는 OS 성능중 중요한 것 중 하나.

프로세스 상태

상태는 프로세스와 자원간의 상호작용에 의해 결정

  1. created state
    작업을 커널에 등록
    PCB 할당 및 프로세스 생성
    커널은 가용 메모리를 체크하여 ready state 로 이동
  2. ready state
    프로세서 외에 다른 모든 자원을 할당 받은 상태
    cpu 만 할당 받으면 실행 가능 상태 running
  3. running state
    프로세서와 자원을 모두 할당 받은 상태
    preemption 을 통해 ready 로 변할 수 있다.
    입출력 자원 대기, block 이 된다.
  4. blocked, asleep
    프로세서 외의 다른 자원을 기다리는 상태 (I/O  등)
    자원 할당은 system call 에 의해 이루어짐
    다시 ready 로 전환된다.
  5. suspended state
    메모리를 할당 받지 못하거나 빼앗긴 상태.
    생성 후, block 후 메모리가 부족할 때
    메모리 상태를 이미지로 찍어서 다시 resume 할 수 있도록 저장한다.
    swap 메모리에 저장한다.
  6. terminated state
    프로세스 수행이 끝난 상태
    모든 자원을 반납 후 커널 내에 일부 PCB 정보만 남아있는 상태.
    이후 프로세스 관리를 위해 정보를 수집한다.

프로세스 상태

메모리 유무에 따라 active, suspended 로 크게 2 개로 나뉜다.

예상치 못한 외부에서 발생한 이벤트

콘솔, 클럭, 시스템 콜, 프로그램 체크 인터럽트들 등..

인터럽트 처리 과정

인터럽트 발생
프로세스 중단
인터럽트 처리
처리 = 인터럽트 핸들링(결정, 판단) -> 인터럽트 서비스(수행)
인터럽트 종료 후 프로세스가 자원을 뺏길 수도 있다.

컨텍스트 스위칭

컨텍스트 - 프로세스와 관련된 정보들의 집합
pcb 가 cpu 의 레지스터, 메모리 등에 저장된다.

  1. context saving
    프로세스의 레지스터 컨텍스트를 저장하느 작업
  2. context restroing
    레지스터 컨텍스트를 프로세스로 복구하는 작업
  3. context switching
    위 둘을 합하면 스위칭이다.
    커널의 개입이 필요하다.

컨텍스트 스위칭 오버헤드

스위칭은 비용이 든다.
스레드 등을 이용해서 스위칭은 줄이는 것이 좋다.
