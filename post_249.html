설치 파일

우분투는 데비안 계열
dpkg 는 설치 파일 패키지를 나타낸다.
윈도우의 setup.exe 와 비슷한 설치 파일
확장명은 .deb 파일이고 이를 패키지라고 부른다.

dpkg

데비안 패키지를 설치하는 유틸
-i [pac.dev] - 패키지 설치
-r [pac] - 설치만 삭제
-P [pac] - 설정파일까지 삭제

dpkg 문제

의존성 패키지 때문에 복잡해진다.
그래서 apt-get 이 등장함.

apt-get

dpkg 의존성 문제를 완전히 해결함.
인터넷을 통해 패키지 리포지토리를 내려받는다.
deb 파일 따로 필요없이 알아서 설치를 해준다.
저장소는 /etc/apt/source.list 에 저장됨.

apt 를 이용한 설치

apt-get install [패키지] - 설치
-y - 사용자 확인에 모두 yes 처리

패키지 목록 업데이트

apt-get update
source.list 저장소 목록 변경되었을때 적용한다.
그리고 패키지들이 리포지토리에서 변경된 것을 인지한다.?

패키지 업그레이드

apt-get upgrade
최신버전이 있는 패키지들을 설치한다.

패키지 삭제

apt-get remove [패키지]

apt-get 작동 방식

  1. 패키지 설치 명령
  2. source.list 파일에 있는 저장소 주소로 설치 관련 패키지 목록 요청
  3. 저장소에서 설치 관련 목록을 유저에게 전달
  4. 동의하면 목록에 해당하는 패키지들을 설치해줌.

패키지 저장소

main - 우분투 공식 지원. 무료
universe - 우분투 공식 지원x. 무료
restricted - 우분투 공식 지원. 유료
multiverse - 우분투 공식 지원x. 유료

패키지 저장소 2

전세계 모든 유저들의 요청을 하나의 사이트에서 감당을 못함.
그래서 미러 사이트가 있다. 전세계에 깔려있다.
미러 사이트는 모두 동기화된다.
변경하면 apt-get update 명령을 해야 적용됨.

패키지 저장소 3

source.list 에 원본 저장소뿐 아니라 update 저장소도 추가해주면 설치된 패키지의 최신 버전도 설치할 수 있다.
apt-get update 는 환경설정만 하는 것.
apt-get upgrade 를 하면 최신 패키지로 모두 업그레이드 한다.
기존의 패키지를 install 하면 해당 패키지만 업그레이드 할 수 있다.
항상 update 후에 upgrade 를 해야 적용이 된다.

파일 압축, 묶기

압축

xz, bz2, gz, zip 등
xz, bz2 압축률 좋음
리눅스는 대상파일이 그대로 압축되어서 원본은 사라진다.

압축 명령

xz - xz 로 압축하거나 푼다. 압축된 파일 확장명도 xz
xz -d - 압축을 푼다.
bzip, gzip 동일
zip, upzip - 윈도우와 호환을 위함.

묶기 명령

리눅스에서 압축과 묶기는 별개의 프로그램으로 수행.
tar - 묶기 명령어이면서 확장명도 묶음 파일도 tar

tar 동작

-c - 묶기 동작
-x - 풀기 동작
-t - 경로확인 동작

tar 옵션

-f - 파일
-v - 과정보이기
-J - tar+xz

tar 기타

보통 cvf, xvf 를 많이 쓴다.
cvfJ 는 묶기와 동시에 압축도 함.
압축푸는 기능은 tar 에서 있는게 아니라 압축 명령과 연동되어 동작한다.
