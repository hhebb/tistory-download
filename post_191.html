서버

출처 - 얄팍한 코딩사전
서버에 대한 정확한 정의를 모른 채 서버라는 단어에 너무 익숙해져 있다.

서버란

  • 서버는 역할의 개념이다.
  • 서비스를 제공해주는 주체이다.
  • 주로 서버 SW 를 지칭한다. (APACHE, NGINX 등)
  • 서버가 깔린 컴퓨터를 서버라고 지칭한다.

하는 일

  • 클라이언트에게 뭔가를 해준다.
  • 파일, 데이터, 신호, 정보를 주거나 받아서 저장해준다.

클라우드 컴퓨팅

On-premise

  • 자체적으로 서버를 두고 운영한다.
  • 전통적인 방식이다.

클라우드

  • 거대한 집단에서 대규모 컴퓨팅 자원을 소유하여 사용자에게 필요한 만큼 자원을 나눠쓰게 한다.
  • 가상 컴퓨팅 기술을 이용하여 사용자에게 나누어 줄 수 있다.
  • 유동적으로 자원을 할당받을 수 있다.

IaaS, PaaS, SaaS

  • 클라우드도 어디까지 서비스 해주는가에 따라 3 가지로 나눔.

IaaS

  • 서비스로 제공되는 인프라
  • 가상 컴퓨터를 제공하는 개념
  • 하드웨어 측면만 관리해준다.
  • 이상의 서버 설치, 관리 및 작업은 사용자가 한다.
  • 클라우드 컴퓨팅에 해당한다.
  • 종류
    AWS
    GCP
    Azure
    Naver Cloud

PaaS

  • 플랫폼이 서비스로 제공된다.
  • 서버 등의 관리까지 해준다.
  • 사용자는 SW 만 만들어서 올리면 된다.
  • 배포만 하면 PaaS 에서 서비스로 돌려준다.
  • 종류
    heroku - 포폴용으로 좋다.
    netlify

SaaS

  • 소프트웨어를 서비스로 제공한다.
  • 아예 SW 까지 만들어서 그냥 사용하도록 한 것.
  • 바로 온라인으로 이용할 수 있다.
  • 종류
    구글닥스
    드랍박스
    SNS
    유툽
