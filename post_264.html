용어 메모 용

  • OS 공부하다가 모호한 개념을 간략 정리

프로세스

  • 코드
  • 데이터
  • 스택
  • 힙
  • 로 구성된다.

컨텍스트

  • 프로세스의 각종 정보 - PCB 와 혼용하는듯?
  • cpu 가 읽는 레지스터(SP, PC) 값과 연관됨.

컨텍스트 스위칭

  • 실행 중이던 프로세스의 정보가 대기상태로 변한 후 메모리에 저장된다.
  • 메모리에서 대기 중이던 프로세스가 실행된다.
  • 즉, 메모리에 있던 것은 레지스터 등 CPU 를 할당받고, CPU 를 빼앗기면 메모리로 내쳐진다.
  • 이 과정이 합쳐지면 스위칭.
  • 스위칭하면 cpu 가 정지하므로 성능에 영향을 준다.

크리티컬 섹션

  • 동시에 작업하면 안되는 곳.
  • 동시에 여러 스레드가 접근하면 원하는 결과를 얻지 못할 수도 있는 곳.
  • mutex 를 위한 공간

스왑 디바이스

  • 디스크를 메모리로 사용하는데, 대기 중인 프로세스를 저장
  • 가상 메모리를 사용할 때 사용하지 않는 페이지/세그먼트를 임시로 저장하는 곳.

가상 메모리

  • 논리적인 메모리 주소를 사용하여 실제 물리적 메모리보다 더 많은 것처럼 사용.
  • 메모리 맵핑 테이블이 꽉차면 페이지 교체 알고리즘에 의해 특정 페이지가 스왑 디바이스로 내쳐진다.

가상

  • OS, SW 는 하드웨어를 가상화한다.
  • 그 이유는 논리적인 솔루션을 적용하여 자원을 효율적으로 관리 및 활용을 할 수 있기 때문이다.

부트

  • 전원을 넣으면 ROM 에 있는 BIOS 가 특정 스토리지의 부트로더를 읽는다.
    ROM 은 펌웨어로, 반영구적이다.
  • 부트 로더가 부트 코드를 불러 들인다.
  • 스토리지의 커널 이미지를 메모리에 올린다.
