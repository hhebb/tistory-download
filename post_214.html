Node 웹

  • js 는 별로 안좋아하는데 npm 생태계는 무시할 수 없다.
  • 그리고 웹을 하려면 어쩔 수 없이 js 를 써야한다.
  • 그래서 수요도 많고 시장도 크다.

express

  • Node 에서 js 로 백엔드를 돌리는 것을 간편하게 해주는 경령화 프레임워크.
  • 기본적으로 클라이언트 요청을 받고 응답을 주는 작업 전반을 도와준다.
  • 여타 mvc 프레임워크들 처럼 http 작업을 추상화해놓음.
  • 노드 특성 상 대규모 요청을 원활하게 처리할 수 있다.
  • npm 에서 글로벌로 설치해서 사용

generator

  • express 의 확장
  • 뼈대를 빠르게 만들 수 있는 모듈
  • MVC 등의 구조를 만든다.
  • 물론 커스텀 할 수 있다.
  • 기본적으로 route, view, public 등으로 나뉜다.
    route 는 라우팅, view 는 렌더링 뷰 문서.
    public 에는 주로 정적 파일들이 저장되어 있다.

라우팅

  • 웹에서 주로 쓰이는 용어.
  • URI 요청 등의 HTTP 요청에 대해 특정 엔드포인트에 대한 응답 방식.
  • 요청이 들어오는 것을 콜백으로 받아 처리하여 라우팅한다.

정적 파일

  • 이미지, 리소스, 문서 파일 등의 정적인 파일들.
  • express.static 함수에 인자로 폴더이름을 건네줘서 클라이언트에게 공급할 준비를 한다.

미들웨어

  • 클라이언트의 요청과 서버의 응답 사이에서 비즈니스 로직을 의미한다.
  • 라우팅 처리에서 req, res 말고 next 라는 변수에 콜백으로 전달된다.
  • 요청 응답 오브젝트 변경, 요청 응답 종료, 다른 미들웨어 호출 등의 작업을 수행한다.
  • 라우팅 하기 전 미들웨어를 먼저 실행하게 된다.

pug

  • 템플릿 엔진.
  • html 변형의 일종.
  • 다른 언어에서도 백엔드에서 템플릿 엔진을 널리 사용한다.
  • 그만큼 범용적인 기술임.
  • node 에서는 템플릿 엔진이 여러개 있는데, pug 가 가장 많이 쓰임.
  • 보통 views 폴더에 문서로 저장되어서 응답시 변형되어 클라이언트에게 제공한다.

socket.io

  • 전송계층 소켓이 아니라 웹소켓 라이브러리다.
  • 웹소켓은 양방향 통신을 가능케하는 일종의 웹 프로토콜이다.
  • 별도의 포트를 사용하지 않고 HTTP 신호를 이용하여 구현된 것이다.
    그래서 브라우저도 웹소켓을 지원하는지 확인해야 한다.
    HTML 5 의 기술이므로 지원 안하는 브라우저도 있다.
  • 비동기 통신을 위한 모듈로서 이벤트 기반 처리에 의존함.
