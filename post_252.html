용어

ORM, ODM

  • 프로그래밍 환경에서 DB 를 사용할 때 더 사용하기 편하게 해주는 라이브러리.
  • 언어의 객체와 DB 의 row 를 매칭시켜서 하는 듯?
  • 편리하기는 하지만 성능은 pure DB 에 비해서 떨어진다고 한다.
  • 대규모 데이터 처리에는 자제해야하는 듯.
  • SQL 계열의 관계형 DB 에서 Object Relational Model 이라고 한다.
  • NoSQL 계열의 DB 에서는 Object Document Model 이라고 한다.
  • node 에서는 mongoose 라는 mongodb 의 ODM 이 지원된다.
  • 물론 그냥 mongodb 라이브러리를 사용할 수 있다.
  • 단 쿼리를 직접 날려야 한다.

그냥 DB

  • ORM, ODM 이라고 명시안되어 있으면 그냥 DB 시스템이다.
  • 쿼리를 문자열로 작성하여 CRUD 를 수행한다고 보면 된다.
  • python 에서 pymongo, node 에서 mongodb 에 해당한다.
  • python 에서도 pymongo-odm 라이브러리를 사용할 수 있는 듯 하다.
