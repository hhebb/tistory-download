html 초급

- a 요소와 id 속성까지

강의 출처

element, attribute (요소, 속성)

둘을 합쳐서 태그라고 한다.
여는 태그 <>, 닫는 태그 </> 로 구성된다.
그러나 여는 태그가 필요없어서 스스로 닫는 태그도 있다. </>
태그는 안에 정보를 감싸기 위한 것인데, 감쌀 정보가 없는 것은 그냥 닫기만 한다.
태그는 태그를 감쌀 수 있다.
들여쓰기를 통해 nested 상태를 확인할 수 있도록 작성하는 것이 일반적이다.

img 요소

그 중 <img /> 태그는 이미지를 첨부한다.
이 태그 안에 정보를 넣기 위해, img 라는 요소에 src 속성과 alt 속성을 추가한다.
<img src="value" alt="value" /> 속성="값" 의 형식으로 작성한다.
정리하면,
<요소 속성="속성값"/>
<요소 속성="속성값">요소 내용</요소>

a 요소

a 요소는 요소가 감싼 텍스트를 누를 시 해당 주소로 이동하게 만든다.
<a href="">링크 설명</a>
href 는 hyper reference 라는 의미의 속성이다.

절대 주소, 상대 주소

a 요소, img 요소는 href, src 등의 속성의 값을 주소로 적을 때가 있다.
절대 주소는 파일의 절대적인 주소의 풀네임.
상대주소는 현재 html 파일이 있는 폴더에서 접근이 가능한 곳에 있을 때, 상대적인 위치로 나타낸것.

head 요소

doctype 은 html 버전을 명시하는 태그(요소)이다.
<!doctype html>
head 요소의 역할은 문서의 메타데이터의 집합이다.
페이지에서 직접 보이지 않는 정보이다.
css, js 파일을 연결하는 부분들도 이 요소안에 있음.

head 안에 있는 요소들

타이틀 태그
<title>타이틀</title>
한 번만 사용되고, 문서의 제목이나 이름을 나타낸다.
브라우저의 탭에 노출됨.
옛날에는 타이틀을 길게 적었는데, 요새 브라우저들은 탭을 이용하므로 중요한 내용만 짧게 작성해야함.
쇼핑몰의 경우엔 현재 선택한 상품이름을 적는 것이 옳다.

메타 요소

<meta 속성="속성값"/>
메타에서 담을 수 있는 종류는 여러 종류가 있다.
각종 검색엔진에서 요구하는 임의의 메타정보들도 있다.
특히, 문자인코딩에 대한 정보 charset 이 있는데, utf-8 을 권장한다.
<meta http-equiv="content-type" content="text/html; charset=utf-8"/>
description 값은 검색엔진에서 요약문을 확인하게 한다.
<meta name="description" content="요약문"/>
keywords 값은 역시 검색엔진에서 키워드를 확인하게 한다.
<meta name="keywords" content="key1, key2"/>
검색엔진 통과. 아래와 같이 하면 검색엔진이 검색을 안 한다.
<meta name="robots" content="noindex, nofollow"/>

링크 요소

주로 css 파일을 연결할 때 사용하는 요소이다.
<link href="" type="" rel="" />
href 속성의 값은 css 파일의 주소를 적어주면 된다.

스타일 요소

링크 요소를 안쓰고 css 파일을 연결하는 방법.
물론 파일을 분리해서 작성하는 것이 좋음.
<style type=""> 내용 </style>

스크립트 요소

js 를 연결할 때 쓰는 요소.
<script type="" src="abs.js"></script> 가 일반적,
<script type="">스크립트</script>
처럼 html 에서 코드를 작성해도 됨.
스크립트 요소는 head 뿐 아니라 body 태그안에 들어가도 된다.

블록과 인라인(body 에 들어가는 요소들)

body 안에 들어가는 요소는 크게 2 가지 요소로 나뉜다.
블록 요소, 인라인 요소

블록 요소

기본적으로 줄바꿈이 일어나는 형태.
한 행 전체를 포함한다.
영역의 너비가 상위 영역의 전체 너비만큼 된다.
h1~h6 요소, div p 요소 등이 있다.

인라인 요소

줄바꿈이 일어나지 않는다.
태그로 감싼 영역만 포함한다.
a, img, strong, span 요소 등이 있다.

제목 요소(헤드라인)

h1~h6 까지 순서대로 작아진다.
주로 가장 중요한 내용은 h1 으로 작성

p, pre, br, hr 특문

p 요소는 그저 본문을 나타내기 위한 태그로 활용하도록 한다.
p 는 줄바꿈이 일어나는 블록 요소임에도 불구하고 다른 블록 요소를 포함할 수 없고
인라인 요소만 포함가능하다.
pre 요소는 띄어쓰기, 줄바꿈, 공백, 들여쓰기 등 문자 그대로를 노출한다.
아스키코드 그림, 시, 소스코드 등을 나타낼 때 사용한다.
역시 블록 요소이지만 블록 요소를 포함할 수 없다.
Br 요소는 줄바꿈 요소다.
디자인 상 강제로 줄을 바꿔야할 땐 css 로 해결하는 것이 맞다.
Hr 요소는 시각적으로 가로선을 긋는다.
그러나 시각적, 디자인적인 부분은 모두 css 로 해결하는 것이 맞다.
특문(escape) 은 몇가지 알아둬야 한다.
띄어쓰기 - &nbsp;
& - &amp;
< - &lt
> - &gt
등등

블록, 인라인 요소 규칙

대부분의 요소는 같은 형태의 요소를 포함 가능
대부분의 블록 요소는 인라인 요소 포함 가능
하지만 인라인 요소는 블록 요소 포함 불가능

Em, Strong 요소

둘 다 강조하는 인라인 요소이지만 Em 은 이탤릭, Strong 은 bold

blockquote, q, cite 요소

모두 인용에 관련된 요소다.
blockquote 는 블럭요소로, 들여쓰기로 표현된다.
긴 내용을 인용할 때 적합.
q 는 인라인 요소로, 따옴표 모양으로 감싸져 나타난다.
cite 요소는 인용된 자원을 나타냄.
제목 등을 마크업한다.
blockquote 나 q 의 속성으로 cite 는 출처의 url 을 넣을 수 있는데, 검색엔진만 알아본다.

ins, del 요소

ins 는 텍스트의 삽입, del 은 텍스트의 삭제
del 은 취소선, ins 는 밑줄
특이하게도 ins 와 del 은 인라인/블록 2 가지 형태로 사용가능하다.
블럭으로서 다른 요소를 포함하기도 하고, 인라인으로서 사용가능하기도 하다.
당연히 인라인으로 쓰일땐 다른 블럭을 포함할 수 없다.

a 요소, id 속성, title 속성

a 는 하이퍼링크 요소였지만 문서 내에서도 링크를 걸 수 있다.
a 의 역할은 링크, 앵커 두가지가 있고, a 는 anchor 의 약자다.
a 는 더이상 앵커 역할을 하지 않고 링크 역할만 한다.
사실 id 요소가 앵커 기능을 수행한다.
<h1 id=""> </h1> 로 h 요소의 속성으로 작성한다.
물론 id 는 같은 문서내에서 중복되지 않아야 한다.
id 는 모든 요소에 쓸 수 있어서 편리하다. css, js 에서도 id 속성 활용도가 높다.
<a href="#id"> </a> 로 유일하게 정의된 id 에 링크를 걸 수 있다.
추가적으로 title 속성은 툴팁으로 부연설명을 해준다. 역시 모든 요소에 추가할 수 있는 속성이다.
a 요소의 href 속성에 팝업을 띄울려면
<a href="" onclick=".js"></a> 등으로 연결한다.

a 요소에 들어가는 추가 속성

target, rel 속성.
둘 다 href 속성과 관련되므로 href 와 함께 사용해야함.
target 속성은 잘 안 쓴다.
rel 속성은 헤딩 링크와의 관계를 나타낼 수 있다.
altenate, author, bookmark, help, license, prev, next, search 등의 속성값이 있다.
이들은 브라우저에 표시는 안되지만 검색엔진을 위해 넣어준다.
