웹 표준

브라우저에 잘 표시되도록 웹표준을 지켜서 html 을 작성해야 한다.
크게 2 가지 html, css, js 를 역할에 맞게 분리하는 것과 각 문서를 표준에 맞게 작성하는 것이 원칙임.
html(의미) css(외형) js(동작)
첫 째로 열고 닫는 태그를 온전히 닫아 주는 것.
그 다음 블록 요소를 인라인 요소에 넣지 않는 것.
ul, dl, p 같은 특수한 경우 신경을 써주기. 자식으로 li 만 쓴다거나 등등.

doctype

표준을 지키는 타입이 몇가지 있으므로 문서의 타입을 지정해줘야 한다.
문서의 가장 먼저 선언해준다.

doctype 버전

xhtml - 더 엄격한 버전
html5 - html 5 버전. 태그를 꼭 안닫아도 된다.
html5 이전의 버전들에서는 doctype 요소에 url 이 들어갔는데 외울필요는 없다.

doctype 종류

표준과 비표준

표준을 지키는 이유는 여러 브라우저에서 같은 결과를 내기 위해서다.
이를 크로스 브라우징이라고 한다.
비표준이면, 브라우저마다 각기 다른 결과가 나오기 때문에 반드시 표준을 지키도록 해야한다.
올바른 독타입을 선언하는것도 중요하다.

사용하지 말아야 할 요소들

과거에는 쓰였지만 지금은 쓰이지 않거나, 표준에 반하는 요소.

절대 쓰면 안되는 요소

외형에 관련된 것은 모두 css 로 넘겨야 한다.
font - 폰트
center - 정렬
blink
marquee

지양해야 할 요소들

iframe - 프레임을 만들어 안에 다른 웹페이지를 보여줄 수 있는 요소.
검색엔진과 접근성이 안좋아 중요한 거라면 사용하지 않는것이 좋다??
big/small - 글자를 크게 해주거나 작게 해준다. 당연히 css 에서 폰트 사이즈로 지정해야함.
i, b, s - 이탤릭, bold, strike 인데 em, strong, del 로 바꿔쓰는게 좋다.

문구 요소들 2

em, strong, quote 등의 문구 요소들 외에 쓰이는 문구 요소들.

abbr 요소

축약어 요소. 축약어에 대한 정보가 들어간다.
<abbr title="풀네임">축약어</abbr>
title 이 있어서 커서를 갖다대면 툴팁에 풀네임이 표시된다.

address 요소

주소가 아닌 연락처 의미를 담고 있다.
페이지의 저자 또는 소유주의 연락처, 이메일 등 연락정보를 담는다.
address 는 블록 요소다.
<address> <p> </p> </address>

dfn 요소

개념의 정의를 나타내는 요소.
dl 요소와 비슷하지만 dt,dd 같은 그룹은 없다.
만약 안에 title 속성이 있다면 그 값이 정의이다.

code, samp 요소

code 는 소스코드를 표현한다.
samp 는 프로그램이 화면에 출력하는 텍스트를 표현한다.
하지만 css 를 쓰는것이 좋다??

테이블 요소의 기본 구조

테이블을 만드는 요소이다.

테이블 요소, tr, td

테이블 요소는 전체 테이블을 표현한다.
tr 은 행을 표현하고 td 는 셀(열) 을 표현한다.
th 는 특별히 title 행을 의미한다.
<table> <tr><td></td></tr> </table>
th, td 에는 모든 요소가 들어가지만 tr 에는 th 나 td 만 들어간다.

colspan, rowspan 속성

테이블에서 셀 병합을 하고 싶을 때 사용
rowspan 은 세로로 행들을 합치고 colspan 은 가로로 열들을 합친다.
<td colspan="2"> </td>
헷갈린다.

테이블 요소의 의미적 작성

테이블을 더 의미있게 만드는 요소와 속성을 알아본다.

caption 요소

표의 제목을 나타낸다.
테이블 요소의 가장 첫 자식으로 와야한다.
<table> <caption> </caption> </table>
제목을 넣는 것은 표준의 요소는 아니지만 접근성에서 필수이다.

summary 속성

table 요소에 넣는 속성. 브라우저에 보이지는 않는다.

colgroup 요소

하나 이상의 열을 그룹짓는데 사용
캡션 다음으로 올 수 있는데 필수는 아니다.

thead, tbody, tfoot 요소

테이블에서 행으로 구조적 의미를 가지게 한다.
colgroup 과 반대로 가로 방향으로 의미를 준다.
thead 는 한 번 만 쓸 수 있고 tbody 나 tfoot 보다 먼저 선언되어야 한다.
<table> <thead><th></th></thead> </table>
tbody 는 여러번 사용된다.
tfoot 은 한 번만 사용된다.
<table> <tbody><th></th></tbody> </table>

scope 속성

반드시 thead 에서만 사용할 수 있는 속성.
th 요소가 어느 셀에 영향을 미치는가를 지정한다.
다시 말하면 th 요소가 어느 영역의 제목을 뜻하는 지 정의한다.
복잡해지므로 주의해야 함.
