SNS api 사용

예전부터 카톡 봇 만들기, SNS, 블로그 자동 로그인 등을 해보고 싶었다.
rest api 도 뭔지 잘 모르고 서버 클라이언트가 잘 모르던 때라서 방대한 양의 공식 문서만 보고 때려쳤었다.
오늘은 할 게 딱히 없어서(할 거 있는데 귀찮아서) 벼르고 벼르던 api 사용을 해보았다.
몇 시간 잡고 있어도 계속 오류 나고 난리나서 포기하려고 했는데 결국 해냈다.
웹 api 의 개념을 이해해야 했다.
카카오도 자신들이 만들어 놓은 기능을 조립해서 카톡 앱을 배포한다.
완전 같진 않지만 비슷한 방식으로 개발자들에게 api 를 제공하면, 개발자들은 그걸로 자신만의 앱을 만든다.
단 어느정도 인증은 필수다.
증권사도 자신들이 만든 api 를 조립해서 HTS 를 만들고 그걸 또 공개한다.
사용자 정보가 요구되는 기능은 로그인의 개념으로 인증을 거쳐야 한다.
그냥 흔히 알고 있는 그 로그인이 맞다.
웹 rest api 는 다 비슷비슷하게 동작한다.
하나 할 줄 알면 다 똑같이 적용할 수 있다.

방식

  • 보통 웹 기반 서비스 로그인은 유사하다.
  • API 로 구현한 앱에서 인증 및 로그인
    • 미리 API 키를 발급 받아놓고 그걸로 인증을 한다.
    • 보통 OAUTH 2.0 혹은 2FA 방식으로 인증한다.
    • 인증은 보통 callback 방식, redirection 방식 둘 중 하나다.
  • 서버에서 토큰 발행 후 사용
    • 토큰 만료 시간이 있는 경우도 있다.
