스마트 컨트랙트

  • 일종의 프로그램
  • 블록체인에 저장되어 있음.
  • 어디에 저장되어 있는지 알기 위해 주소를 쓰는데, 어카운트로 취급된다.
  • 특정 주소에 배포되어 있는 트랜잭션으로 실행 가능한 코드.
    • 맛컨 소스코드는 상태와 함수로 구성됨.
    • 소스코드는 블록체인에 저장됨.
    • 함수는 상태를 변경하는 함수, 그렇지 않은 함수로 분류.
    • 사용자가 맛컨 함수를 실행하거나 상태를 읽을 때 주소가 필요.

Solidity

  • 이더리움 클레이튼에서 지원하는 맛컨 언어
    • 스크립트 언어라서 컴파일 안함.
  • 제약이 있는데 포인터 개념이 없다.
  • 컨트랙트는 Code(함수) + Data(상태)
  • public 만 실행 가능
  • view(pure) 는 상태를 변화시키지 않는 순수함수라는 뜻.
  • 반환값이 있으면 반환 타입도 명시한다.

Solidity 2

  • contract 키워드로 컨트랙트를 정의.
  • 일종의 클래스처럼.

블록체인 어플리케이션

  • 모바일에서는 암호화폐 결제를 못하게 한다.
  • 그래서 웹 기반 블록체인 앱이 발달했다.
  • 이더리움 중 wallet 중 하나가 크롬의 익스텐션.

wallet

  • 보통 지갑은 어카운트를 표현하는 것
  • 브라우저, 플러그인, 하드 등에서 접근 가능한 것
    • 잔고 표시
    • 토큰 전송
  • 클레이튼 SDK 는 in-memory wallet 가 있다.
    • 임시로 생성되고 사용되는 어카운트??

클레이튼 SDK

  • 솔리디티로 만든 스마트 컨트랙트를 배포하는 데 사용하는 웹 SDK!
  1. in memory wallet 생성 - 2 개 생성해서 from, to 로 사용
  2. tx 생성 - from 에서 to 로 ~ 를 할 것이다.
  3. 서명 - 비밀키로 서명
  4. 전송 - 비동기
  5. 배포(bytecode + ABI) - 솔리디티 컴파일 결과를 블록체인에 추가
  6. 맛컨 실행 - 트랜잭션을 발생시켜서 컨트랙트를 실행한다. 가스비도 적음?
