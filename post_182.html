네이티브 객체

  • ES 명세에 정의된 객체이며 앱 전역의 공통 기능을 제공한다.
  • 앱 환경의 제약없이 언제나 사용가능.
  • 기본 자료형 객체, Date, Math 등의 객체 생성과 관련있는 함수들.
  • 이는 전역 객체와 구분해야한다!

Object

  • var o = new Object();
  • 잘 안쓰이는 방법

Function

  • 모든 함수는 객체다.
  • new Function() 으로 함수 객체를 만들 수 있다.

Boolean

  • boolean 래퍼 객체이다.
  • 주의할 것은 불리언 객체는 객체이므로 평가식에서 항상 참으로 간주한다는 것이다.

Error

  • error 객체를 생성한다.
  • 런타임 에러가 발생했을 때 throw 된다.
  • throw new Error();

원시 타입과 래퍼 객체

  • 모든 원시 타입은 래퍼 객체의 메서드를 사용할 수 있다.
  • 이는 메서드 호출 시 원시 타입 값은 래퍼 객체로 일시적으로 변환되기 때문임.

호스트 객체

  • 브라우저 환경에서 제공하는 window, XmlHttpRequest 등의 DOM 노드 객체 등을 말한다.

전역 객체

  • 모든 객체의 유일한 최상위 객체.
  • 브라우저에서는 window, 서버에서는 global 객체를 의미한다.

BOM (Browser Object Model)

  • 브라우저의 탭 또는 창의 모델을 생성한다.
  • 최상위 객체는 window 객체로 현재 브라우저 창 또는 탭을 표현
  • window 의 자식으로 document, history, location, navigator, screen 등이 있다.
  • 표준 내장 오브젝트가 구성된 후에 구성된다.

DOM (Docement Object Model)

  • 현재 웹페이지의 모델을 생성한다.
  • 최상위 객체는 document 객체로 전체 문서를 표현한다.
  • html 태그의 구조를 가진다.
  • 표준 내장 오브젝트가 구성된 후에 구성된다.

Global Object

  • 실행 컨텍스트에 컨트롤이 들어가기 전에 생성이 된다.
  • constructor 가 없으므로 new 로 새롭게 생성할 수 없다.
  • 전역 스코프를 갖는다.
  • 전역 객체의 자식 객체를 사용할 때, 전역 객체 기술은 생략할 수 있다.
    예를 들어 window.document 는 document 라고 표기한다.
  • 전역 객체는 전역 변수를 프로퍼티로 가지게 된다.
  • 글로벌 영역의 함수는 전역 객체의 메서드다.
  • 사실상 모든 객체의 부모

전역 프로퍼티

  • 전역 객체의 프로퍼티
  • 앱 전역에서 사용하는 값들을 위해 사용.
  • 전역 프로퍼티는 간단한 값이 대부분이다.
  • 기본적으로 Infinity, NaN, undefined 등도 있다.

전역 함수

  • 기본적으로 eval(), inFinite(), isNaN() 등이 있다.
  • parseFloat()
  • parseInt('_', 진수)
  • encodeURI(URI), decodeURI(encoded)
    네트워크 통신할 때 문자열을 이스케이프 처리하는 것이다.
     발생할 수 있는 문제를 방지하기 위해서 사용한다.

Number 래퍼 객체

  • new Number(item) item 이 숫자로 변환할 수 있으면 숫자 객체를 반환한다.
  • Number(item) 처럼 new 를 사용하지 않으면 객체 생성 안하고 숫자 값을 반환한다.

Number 프로퍼티

  • Number.EPSILON - ES6. 가장 작은 수
    부동 소수점은 정확하지 않아서 두 수의 차이와 EPSILON 과 비교해야 한다?
  • Number.MAX_VALUE - 가장 큰 수. 이보다 큰 수는 Infinity
  • Number.MIN_VALUE - 표현가능한 가장 작은 수. 이보다 작은 수는 0
  • Number.POSITIVE_INFINITY - 양의 무한대
  • Number.NEGATIVE_INFINITY - 음의 무한대
  • Number.NaN - Not a Number. window.NaN 과 같다.
  • Number.MIN_VALUE - 가장 작은 수. 이보다 작은 수는 0

Number Method

  • Number.isFinite(val) - 유한수인지 검사하여 boolean 반환
    전역함수 isFinite 는 형변환하지만 얘는 안함. 그래서 숫자가 아닌것이 들어오면 항상 false
  • Number.isInteger(val) - 정수인지 검사하여 boolean 반환. 역시 형변환은 안한다.
  • Number.isNaN(val) - NaN 검사하여 boolean 반환
  • Number.prototype.toExponential(digit) - 지수 표기법을 문자열로 반환
  • Number.prototype.toPrecision(digit) - 반올림 함수, 문자열로 반환
  • Number.prototype.toString(digit) - 숫자을 문자열로 변환하여 반환

Math 객체

  • 수학 상수, 함수 를 위한 프로퍼티와 메서드를 제공하는 빌트인 객체
  • 생성자 함수로 된 객체가 아니라서 static 요소만 있다.

Math 프로퍼티

  • Math.PI

Math 함수

  • Math.abs(num)
  • Math.round(num) - 반올림
  • Math.ceil(num) - 천장 함수
  • Math.floor(num) - 바닥 함수
  • Math.sqrt(num) - 제곱근
  • Math.random() - 0 ~ 1 실수 반환. 이 값을 조작하여 다양한 범위의 값을 얻는다.
  • Math.pow(x, y)
  • Math.max(num)
  • Math.min(num)

Date 객체

  • 시간을 위한 빌트인객체
  • 생성자 함수.

Date Contructor

  • 생성자 함수이므로 객체를 생성할 수 있다.
  • 생성 방식은 4 가지
    1. new Date()
    2. new Date(millisec
    3. new Date(dateaString)
    4. new Date(year, month, ...)

Date 메서드

  • Date.now() - 현재 시간까지 경과한 밀리초 숫자를 반환. 타임스탬프 개념?
  • Date.parse(date) - 특정 시각에 대한 밀리초를 반환. 타임스탬프
  • Date.UTC(date) - parse 와 비슷하지만 숫자들을 넣어야함.
  • Date.prototype.getFullYear() - Date 객체에서 년도 4 자리 반환. month, day, 이하에서도 가능
  • Date.prototype.setFullYear() - Date 객체에서 년도 4 자리 설정. month, day, 이하에서도 가능
  • Date.prototype.toDateString - Date 객체를 사람이 읽을 수 있는 문자열로 날짜 반환
  • Date.prototype.toTimeString - Date 객체를 사람이 읽을 수 있는 문자열로 시간 반환
