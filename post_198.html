Rest API

출처 - 얄팍한 코딩사진
기술이 아니라 형식이다. 준수하는 것이 좋다.

개념

  • HTTP 에서 요청 및 응답에 사용되는 인터페이스. 형식.
    HTTP 통신에 어떤 URI, 어떤 메서드를 사용할지에 대한 약속!
  • URI 형태로 이루어지는데, 명사로 나타내야한다.

특징

  • 요청 그 자체로 어떤 의미인지 알 수 있다.
  • 기능 뿐만 아니라 의미까지 명시하여 누구나 쉽게 API 처럼 쓸 수 있다.

메서드

  • HTTP 요청의 행동에 해당하는 것. C.R.U.D
  • 아래 중 하나만으로 모든 일을 할 수도 있지만, 의미를 명확하게 하기 위해 목적에 맞게 쓴다.
  • URI 는 명사로 이루어지고 동사형인 메서드를 함께 사용하게 된다.
  • 종류
    GET - 조회
    POST - 추가
    DELETE - 삭제
    PATCH, PUT - 갱신. PUT 은 통쨰로 갈아끼울 때 사용.
