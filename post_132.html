운영체제 구분

동시 사용자 수 - 싱글, 멀티 유저 시스템
동시 실행 프로세스 수 - 싱글 태스트, 멀티 태스트 시스템
작업 수행 방식 -

동시 사용자 수

  1. 단일 사용자 - 한 명 사용자만 동시에 사용 가능
    한 명이 독점한다.
    비교적 간단하고 시스템 보호 방식이 간단
    개인용 pc, mobile
    윈도우, 안드로이드, 도스 등
  2. 다중 사용자 - 동시에 여러 사용자들이 시스템 사용.
    시스템 자원, 파일 들에 대한 소유 권한 관리가 필요하다.
    os 가 복잡하다.
    서버, 클러스터, unix, 리눅스, 윈도우 서버 등

동시 실행 프로세스(프로그램) 수

단일 작업 - 시스템 내에 하나의 작업만 존재
하나의 프로그램 실행 마치고 다른 프로그램 실행
OS 구조가 간단
DOS 등이 있다.
다중 작업 - 동시에 여러 작업 수행가능
OS 기능 및 구조가 복잡
유닉스, 리눅스, 윈도우 등이 있다.

작업 수행 방식

  1. 순차 처리 - 옛날에 OS 가 없을 떄. 직접 기계어로 프로그램 작성
  2. 배치 시스템 - 사용자 요청 작업 등을 모아 일괄적으로 처리. 엄청 옛날에 쓰임. 전자계산소 등
    시스템 지향적.
    처리 효율 향상되지만 생산성 저하, 긴 응답시간
  3. (time sharing system) 시분할 시스템 - 여러 사용자가 자원을 동시에 사용
    OS 가 파일 시스템 및 가상 메모리를 관리
    사용자 지향적이다.
    대화형, 단말기 등을 사용.
    응답시간이 짧고 생산성이 향상되었지만 통신비용이 올라가고 보안 문제가 생김.
    그리고 동시 사용자가 많아지만 체감 속도가 느리게 느껴진다.
  4. 개인 컴퓨터 - 개인이 시스템을 독점함
    cpu 활용률이 고려 대상이 아님.
    os 가 상대적으로 단순하지만 사용자 지원 기능을 지원함.
    편리성을 극대화함.
    빠른 응답을 가지지만 성능이 낮다.
  5. 병렬 처리 시스템
    단일 시스템 내에서 둘 이상의 프로세서 사용
    메모리 등의 자원을 공유한다. - tightly coupled system
    성능을 높이기 위해, 신뢰성을 향상하기 위해 사용
    프로세서간 관계 및 역할을 관리할 필요가 있다.
  6. 분산 처리 시스템
    네트워크를 기반으로 구축된 병렬처리 시스템
    네트워크를 이용하여 여러대의 컴퓨터를 붙인 효과
    loosely coupled system
    물리적인 분산, 통신망을 이용해 상호 연결한다.
    각각 OS 를 탑재한 다수의 범용 시스템으로 구성
    분산 운영체제를 통해 하나의 프로그램, 자원처럼 사용가능하다.. - 은폐성
    클러스터 시스템, 슈퍼컴, 클라이언트-서버 시스템, P2P
    장점은 자원공유를 통해 높은 성능을 가지며 신뢰성이 높다.
    단점은 구축과 관리가 어렵다.
  7. real-time 시스템 - 작업처리에 제한시간을 갖는 시스템
    제한 시간 내에 서비스를 제공하는것이 자원활용 효율보다 중요하다.
    hard real-time task
    시간 제약을 지키지 못하면 치명적일 때 - 발전소 제어, 무기 제어 등
    soft real-time task
    영상 재생 등

OS 의 구조

  1. 커널 - 알맹이
    OS 의 핵심 부분(메모리에 상주한다)
    가장 빈번하게 사용된다.
    프로세서, 메모리 등 시스템을 관리한다.
    상주 프로그램, 관리자 등이라고도 한다.
  2. 유틸리티
    비상주 프로그램
    UI 등 서비스 프로그램
    [hw - 커널 - 시스템 콜 - 유틸 - 앱 프로그램] 에서 유틸까지가 OS 이다.

단일 구조

모든 기능을 하나의 커널에 모두 집어넣은 구조
장점 - 커널 모듈 간 직접 통신하여 효율적으로 자원관리 가능
단점 - 커널의 거대화가 되어 유지보수가 어렵다.

계층 구조

모듈화.
기능 별로 계층을 나눈다.
단일 구조 대비 성능이 저하된다. 작업을 하는데 계층을 타야하기 때문임.

마이크로 커널

커널이 너무 커지다 보니 아예 필수 기능만 포함.
커널의 크기를 최소화.
기타 기능은 사용자 영역에서 수행

OS 의 기능

프로세스 관리, 메모리 관리, 프로세서 관리, 파일 관리, 입출력 관리, 주변장치 관리 등

프로세스 관리

  1. 프로세스
    커널에 등록된 실행 단위
    아용자 요청, 프로그램의 수행 주체(entity)
  2. OS 프로세스 관리 기능
    생성/삭제, 상태 관리, 자원 할당, 교착상태(데드락) 해결, 프로세스 통신 동기화
  3. 프로세스 정보 관리(PCB)

프로세서 관리

  1. CPU
  2. 스케쥴링
  3. 프로세서 할당 관리

메모리 관리

  1. 주기억장치 - 작업을 위한 프로그램 및 데이터르 올려놓는 공간
  2. 멀티 유저, 멀티 태스킹 시스템
    각 프로세스 할당 메모리 영역 접근 보호
    프로세스에 메모리 할당 및 회수
    메모리 여유공간 관리

파일 관리

  1. 파일 - 논리적 데이터 저장 단위
  2. 사용자 및 시스템 파일 관리

입출력 관리

장치들은 직접 CPU 접근 못하므로 OS 를 반드시 거치게 된다.
