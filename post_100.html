다양한 상황에서 정렬을 한다.
사실 그냥 sort 쓰는게 거의 정답이다.
하지만 정렬에 대한 지식은 모두 갖고 있어야 한다.

수 정렬하기 2

  • 난이도 - 하

  • 유형 - 정렬

  • 핵심
    사이즈 확인.
    백만이므로 $O(NlogN)$ 이면 가능하다. 백만의 log2 는 약 20.

  • 비고
    퀵 정렬, 힙 정렬, 병합 정렬 을 작성해서 사용해도 됨.
    퀵정렬은 최악의 경우 효율이 떨어지므로 병합 정렬이나 힙 정렬 추천.
    백준에서 메모리 허용치가 꽤 높다면 pypy 로 제출하기.
    2의 10 제곱은 1024 = 1000

    병합 정렬을 하나의 함수로 만들기.
    단순히 알고리즘을 그대로 외우는 것을 넘어서 로직을 합치고 나누는 과정을 자유롭게 할 수 있어야 한다.
    다른 알고리즘도 마찬가지다.

# 수 정렬하기 2
import sys

n = int(sys.stdin.readline())
arr = []
for _ in range(n):
    arr.append(int(sys.stdin.readline()))#int(input()))

def merge(left, right):
    pr, pl = 0, 0
    merged = []

    while pr < len(right) and pl < len(left):
        if left[pl] > right[pr]:
            merged.append(right[pr])
            pr += 1
        else:
            merged.append(left[pl])
            pl += 1

    while pr < len(right):
        merged.append(right[pr])
        pr += 1

    while pl < len(left):
        merged.append(left[pl])
        pl += 1

    return merged


def split(arr):
    if len(arr) <= 1:
        return arr

    left = arr[:len(arr)//2]
    right = arr[len(arr)//2:]

    return merge(split(left), split(right))

arr = split(arr)

for i in arr:
    print(i)
# 수 정렬하기 2 - merge 와 split 합친 함수.
import sys

n = int(input())
arr = []
for _ in range(n):
    arr.append(int(input()))#int(input()))

def merge_sort(arr):
    if len(arr) <= 1:
        return arr

    left = merge_sort(arr[:len(arr)//2])
    right = merge_sort(arr[len(arr)//2:])

    pr, pl = 0, 0
    merged = []

    while pr < len(right) and pl < len(left):
        if left[pl] > right[pr]:
            merged.append(right[pr])
            pr += 1
        else:
            merged.append(left[pl])
            pl += 1

    while pr < len(right):
        merged.append(right[pr])
        pr += 1

    while pl < len(left):
        merged.append(left[pl])
        pl += 1

    return merged

arr = split(arr)

for i in arr:
    print(i)

K 번 째 수

  • 난이도 - 중
  • 유형 - 정렬
  • 핵심
    갯수가 많으므로 O(N) 의 계수정렬은 못쓴다.
    이왜중?
    5,000,000 의 nlogn 은 1 억이라서 조건이 빡빡함.
    어쨌든 문제풀 때 고급 정렬을 직접 구현할 일은 거의 없다.
    하지만 반드시 직접 작성할 수 있어야 한다.
# K 번째 수
n, k = (map(int, input().split()))
arr = []
arr = list(map(int, input().split()))
arr.sort()
print(arr)
print(arr[k-1])
