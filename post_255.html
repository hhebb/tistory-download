서버 운영

  • 옛날부터 원격 접속은 많이 활용했다.
  • 고성능 서버에서의 작업을 개인용 pc 에서 수행하는 것.
  • 네트워크를 이용해 원격 접속을 한다.
  • 원격은 필수적으로 필요.

텔넷

  • 텔넷 서버를 설치하면 텔넷 클라이언트 프로그램이 있어야 함.
  • 서버는 텔넷 소켓을 열어놓는다.
  • 클라이언트는 텔넷 서버에 접속을 요청한다.

구축 과정

  • 서버 설치
  • 설정 파일 편집. - text file
  • 사용자 생성 - adduser. 리눅스 사용자 개념
  • 텔넷 서비스 데몬 가동 - systemctl 로
  • 방화벽 설정 - 기본적으로 incoming port 가 막혀있다. port 를 개방해야 한다. ufw 로
  • 클라이언트 접속

텔넷 주의

  • root 사용자로 접속은 보통 막아놓음
  • 필요하다면 /etc/securetty 를 지우면 된다.
  • 그냥 일반계정으로 들어가서 root 권한 획득하면 된다.
    서버는 역할이다. 그리고 프로그램, 프로세스, 데몬이다.
    주체는 컴퓨터다.

ssh 서버

  • 보안 강화된 원격 접속 서버

구축 과정

  • ssh 서버 설치
  • 유저 생성 - adduser
  • 방화벽 설정 - ufw allow 22/tcp
  • 클라이언트 접속 - 사용자@서버, 암호만 있으면 됨.

상시 가동

  • 껐다 켜도 자동으로 재가동
  • systemctl enable ssh

VNC 서버

  • Xwindow 모드를 원격으로 가동
  • 그래픽 데이터를 전송하므로 조금 느림.

DB 서버

  • db 를 위한 서버 운영하는 경우도 매우 흔하다.

DB

  • dbms - db 들을 관리
  • db - 테이블들이 있음
  • table - 데이터들이 표로 되어있음
  • SQL - dbms 다루는 언어

db 서버 접속

  • db 서버에 직접 가서 작업은 잘 안한다.
  • 보통 원격으로 작업하니까.
  • 기본적으로 외부 원격은 막아놓는다.
  • db 가 존재하는 곳이 db 서버, 원격에서 접속하는것을 db 클라이언트
  • 클라이언트는 os 에 접근하고 거기서 db 에 또 접근해야 한다.
  • db 들도 데몬으로 돌아가는 또 하나의 서버 프로그램이다.
  • 각 db 별로 포트가 있어서 개방해줘야 한다.
  • 사용자는 db 에서 db 사용자를 따로 추가해줄 수 있다.
  • 서버(서비스, 데몬) 은 보통 d 를 붙인다. mysqld, mongod

db 계정

  • 서버 계정 말고 db 의 계정도 있다.
