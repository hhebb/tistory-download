GIt 과 GitHub 맛스타

꽤 오래전부터 깃허브를 저장소로 이용했다.
하지만 정말 저장하는 용도로만 쓰고 다른 기능은 쓰지도, 알지도 못했다.
협업 비슷한것을 하면서 사용하기도 했지만, 편리한 데스크탑 앱만 사용해서 진면목을 알지 못한다.
깃이 뭔지 모르고 그냥 깃허브만 깔짝댄것임.
Git 이 뭔지, 어떻게 터미널에서 잘 쓰는지 정리를 할 것이다.

Git

툴
버전관리, 형상관리 툴
로컬 저장소.
깃과 깃허브는 다른것이다.

GitHub

깃을 사용하는 클라우드 서비스
리모트 저장소.
이외에도 깃랩, 깃버킷 등의 저장소도 있다.

용어

  • origin
    원격 저장소 이름
    자동으로 원격 저장소에 origin 이 붙는다.
  • master
    브랜치 중 가장 중심이 되는 브랜치
  • stage area
    커밋하기 전 대기 공간
  • working directory
    작업 폴더
  • head
    커밋이 된 상태의 공간
  • repository
    실제 소스코드
    커밋 내역, 스냅샷
    각종 메타데이터, 설정파일
    해시 값들

깃 명령어

  • git config --global user.name _name_ - 이름 등록

  • git config --global user.email _mail_ - 이메일 등록

  • git clone - 원격 저장소에서 로컬 저장소로 가져오기.

  • git remote add ~- 처음 리모트와 로컬을 연결할 때. 리포를 만들면 깃허브 안내문에 포함됨.

  • git init - 해당 폴더에 빈 저장소 생성. 다시 하면 reinit 이 된다.

  • git status - 저장소 상태.

  • git add . - 임시 저장소에 잠시 파일들을 저장.

  • git commit -m "desc" - 커밋. -m 옵션은 설명.

  • git log - 로그 확인

  • git reset _serial --hard - 과거로 돌아가기. 파괴

  • git revert _serial - 과거로 돌아가지만 현재 시점으로 돌아갈 수 있다.

  • git log - 커밋 내역 확인

  • git amend - commit 내역 수정

  • git fetch - 원격 -> 로컬로 다운로드

  • git push - 원격 저장소에 push. -f 명령어는 강제로 push

  • git pull - fetch + merge

  • git branch _branch_name - master 에서 새로운 branch 생성.

  • git checkout _branch_name - branch 로 이동한다. 새로운 branch 는 master 를 그대로 가져온다.

  • git checkout master - master 로 다시 이동한다.

  • git merge _branch_name - branch 를 master 에 병합한다. conflict 주의

  • git rebase _branch - merge 대신 깔끔하게 재배치한다.

  • git branch -D _branch - 안 쓰는 로컬의 브랜치를 삭제한다.

  • git remote - 원격 저장소(깃헙 등) 를 확인한다. -a 옵션은 로컬, 원격 모든 branch 를 확인한다.

  • git push -u - 원격 저장소에 push

  • git clone _url - 원격 저장소의 저장소를 로컬에 내려 받는다.

  • git fetch - 원격 저장소와 싱크를 맞춘다.(동기화).

깃허브 연동 관련

  1. ssh 인증
    별도의 설정없이 원격 저장소와 바로 통신이 가능한 듯.
    ssh 키 만들고 깃허브에 등록을 해야 함.
  2. https 연결
    이 연결을 더 권장함.
    간편하기도 하고 보안도 되어있다.
