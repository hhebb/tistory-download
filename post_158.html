npm, npx

node 레지스트리

js 패키지들이 저장되어있는 거대한 라이브러리 집합소.
pip 처럼 네트워크를 통해 여기서 필요한 라이브러리를 설치할 수 있다.

npm

node.js 에서 js 패키지들을 관리하는 툴이다.
파이썬의 pip 와 유사하다고 생각할 수도 있는데, 사실 그 이상의 개념이라고 봐야한다.
pip 는 install 매니저에 불과하다.
npm 은 install 은 물론 프로젝트 구동, 설정 등 여러가지를 해주는 듯하다.

npm -g

g 옵션을 주지 않고 모듈울 설치하면 현재 프로젝트 폴더에 설치가 된다.
그런데 g 옵션을 주면 자신의 디바이스에 전역으로 설치되어 어디서든지 접근이 가능하게 된다.
의존성이 뒤죽박죽되는 것을 방지하기 위해 그냥 로컬 프로젝트에 저장하는것이 좋을것 같다.

npx

npx 는 레지스트리에서 최신 버전의 모듈을 설치하고 실행한다.
그리고 작업이 끝나면 사라진다.
그래서 일회성 명령이라고 하는 듯하다.
장점은 저장공간을 낭비하지 않을 수 있고, 항상 최신 버전으로 작업을 할 수 있다는 점.
그리고 이것저것 설치해서 관리가 복잡해지는 것을 방지할 수 있다고 함..
